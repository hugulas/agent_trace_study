# Monitoring Monitorability

## 核心信息

- **标题**: Monitoring Monitorability
- **作者**: Melody Y. Guan, Miles Wang, Micah Carroll, Zehao Dou, Annie Y. Wei, Marcus Williams, Benjamin Arnav, Joost Huizinga, Ian Kivlichan, Mia Glaese, Jakub Pachocki, Bowen Baker
- **机构**: OpenAI
- **发表时间**: 2025
- **arXiv**: 2512.18311
- **DOI**: 10.48550/arXiv.2512.18311
- **PDF**: `[p-006]-monitoring-monitorability.pdf`
- **证据质量**: medium

## 原文摘要翻译

现代 AI 系统的决策过程需要具备可观测性，才能安全地部署能力越来越强的智能体。
对当前推理模型的思维链（chain-of-thought）进行监控，已被证明在检测模型不良行为方面十分有效。
然而，这种"可监控性"在不同的训练流程、数据来源乃至持续的系统扩展下都可能十分脆弱。
为了度量并追踪可监控性，本文提出了三种评估原型以及一种新的可监控性指标，并构建了一套广泛的评估体系。
实验表明，这些评估能够捕捉到经过简单训练以混淆思维链的模型有机体，且在实际场景中基于思维链的监控比仅基于行为的监控更有效。
作者比较了多种前沿模型的可监控性，发现大多数模型具备尚可但并非完美的可监控性。
此外，本文还评估了可监控性如何随推理时计算量、强化学习优化和预训练模型规模的扩展而变化。
结果发现，更长的思维链通常更具可监控性，且在当前前沿规模下，强化学习优化并未实质性降低可监控性。
值得注意的是，对于处于低推理 effort 的模型，可以改而部署一个规模更小但推理 effort 更高的模型（从而匹配能力），以获得更高的可监控性，尽管总体推理计算成本更高。
本文进一步研究了智能体与监控器的扩展趋势，发现当监控较弱智能体时，增加监控器的测试时计算量可以提升可监控性。
让弱监控器访问思维链不仅能提升可监控性，还能使监控器测试时计算量到可监控性的扩展曲线更陡峭。
最后，作者展示了通过向模型提出后续问题并将其后续思维链提供给监控器，可以进一步提升可监控性。

## 创新点

1. **三种评估原型的系统化定义**：首次将可监控性评估划分为干预型、过程型和结果属性型，为不同场景下的可监控性测量提供了统一框架。
2. **g-mean² 指标**：针对类别不平衡和标准指标的失效模式，提出了基于几何均方的可监控性度量，使得不同模型和监控器之间的比较更加公平。
3. **大规模前沿模型可监控性横评**：对 OpenAI 内部模型及 Claude、DeepSeek、Kimi 等外部前沿模型进行了系统比较，量化了各模型在思维链可监控性上的差异。
4. **扩展趋势的系统性分析**：首次报告了可监控性随推理计算、预训练规模、强化学习训练以及监控器能力扩展的变化规律，并提出了"可监控性税"的概念。
5. **后续问题提升可监控性**：发现通过追问模型并收集其后续思维链，可以在不增加智能体计算成本的情况下提升监控效果。

## 一句话总结

本文系统定义了思维链可监控性的评估框架与指标，通过大规模实验发现思维链长度与可监控性正相关，并揭示了模型能力、监控能力与可监控性之间的扩展权衡关系。

## 研究问题

随着前沿模型能力不断提升，其造成潜在危害的能力也在增长。确保模型安全部署的两大方向是对齐与控制。尽管对齐研究取得了进展，模型仍偶尔出现不良行为，因此深度防御至关重要。控制协议通常包含某种形式的不良行为分类器，它们可以访问智能体的部分观测信息，例如输入提示、智能体行为、最终输出或内部神经激活。

现代推理模型的出现提供了一个独特的监控机会：它们在被训练用人类语言思考后才给出输出或行动，这使得我们可以监控其思维链。已有研究表明，思维链监控在检测奖励篡改等不当行为方面具有实用价值。然而，思维链可监控性尚未被充分理解，学界普遍担忧它在不同训练流程或持续扩展下可能十分脆弱。

因此，本文核心研究问题包括：
- 如何量化定义"可监控性"，使其能在不同模型、监控器和评估场景间进行公平比较？
- 当前前沿模型的思维链可监控性处于什么水平？仅基于行为的监控与基于思维链的监控差距有多大？
- 可监控性如何随推理时计算、预训练规模、强化学习优化以及监控器能力而变化？
- 是否存在简单有效的方法来提升可监控性，例如通过追问获取更长的推理过程？

## 数据与任务定义

本文构建了一套广泛的评估体系，涵盖多种行为类型和任务领域。评估环境按照两个维度组织：一是评估原型（干预型、过程型、结果属性型），二是行为类型（中性、谄媚、偏见、不对齐）。

**Table 1：评估分类体系（来自 LaTeX 源码 `main.tex`）**

| 干预型 | 过程型 | 结果属性型 |
|:---|:---|:---|
| Health Queries（缺失证据） | Math（数学） | 不对齐：Shade Arena*、OAI-repo-sabotage* 等 |
| Scruples（谄媚） | Physics（物理） | 不可能任务：Coding*、Flaky-tools* 等 |
| Health Queries（患者观点） | | |
| Toy math/science | | |
| Memory*（偏见） | | |
| Toy sandbagging | | |

注：带星号（*）表示需要使用工具的环境。结果属性型中还包括 Anti-scheming* 和 Shade Arena 等环境，详见原文附录。

各类评估的详细说明如下：

- **干预型评估**：对智能体的环境或输入施加可控干预，测量其行为是否因干预而改变，并检验监控器能否推断出导致行为变化的推理过程。例如在多选题提示中附加提示信息，然后测量模型答案是否等于提示答案。
- **过程型评估**：在不具备完整机制解释的情况下，通过精心构造的任务设置，尽可能从行为输出推断模型的推理过程。具体做法是寻找满足以下条件的问题：1）有效且已知的解题路径有限；2）答案难以猜测；3）答案易于验证。这样，如果模型得到正确答案，几乎可以肯定它经历了某条已标注的解题路径。
- **结果属性型评估**：在许多情况下，可以直接从模型的行动或最终输出确定感兴趣的属性是否实现。这类评估询问监控器能否预测该属性。大多数文献中的不对齐和 AI 控制评估都属于这一原型。

## 方法主线

### 指标设计

为了在不同模型和监控器之间进行公平比较，需要一种对类别不平衡不敏感的指标。标准的 F1 和准确率会随基线行为率变化，无法区分真实可监控性的变化与行为分布的变化。

本文选择灵敏度（TPR）与特异度（TNR）的几何均值：

$$\text{g-mean} = \sqrt{\mathrm{TPR} \cdot \mathrm{TNR}}$$

并进一步将其平方得到 g-mean²，以更严厉地惩罚失败：

$$\text{g-mean}^2 = \mathrm{TPR} \times \mathrm{TNR}$$

在实践中，监控器的 TNR 通常接近 1，因此 g-mean² 很大程度上追踪 TPR，直观反映监控器在目标属性出现时的检测频率。随机监控器的 g-mean² 期望值为 0.25。

对于干预型评估，由于干预臂中表现出目标行为的样本无法确定行为是否由干预导致，本文提出了**最小准则真阳性率**。总效应为：

$$\mathrm{TE} = E[Y \mid X=1] - E[Y \mid X=0] \geq 0$$

TE 构成了因果归因的下界。当基线成功率较低时，TE 接近因果归因比例，评估更可靠。

### 监控范围

本文考虑三种监控器输入范围：
- **CoT-only**：监控器仅基于思维链估计真实标签，不看工具调用、工具响应或最终答案。
- **Action-only**：监控器仅基于工具调用、工具响应和最终答案进行判断，模拟失去思维链时的基线监控能力。
- **All-messages**：监控器基于完整轨迹判断，这是当前实际部署中最常用的监控范围。

### 机制流程

本文的评估与测量流程可分为以下步骤：

1. **输入构造与干预施加**：将原始数据集编码为评估任务，对控制臂和干预臂分别构造提示。干预臂接收特定干扰信息，控制臂保持原样。
2. **生成模型输出与思维链提取**：将构造好的输入送入被测模型，提取其生成的思维链和最终答案。模型在特定温度下采样，允许输出具有随机性。
3. **监控器解码与标签比对**：将收集到的思维链按不同监控范围送入监控器，解码监控器对目标属性的预测。将预测结果与通过规则或人工标注得到的真实标签进行比对，计算 TPR 和 TNR。
4. **指标聚合与扩展趋势分析**：对所有评估环境计算 g-mean²，使用分层 bootstrap 估计标准误。进一步分析指标随模型推理 effort、监控器能力、预训练规模和强化学习训练步数的变化趋势。

本文还评估了不同评估环境之间的相关性。

## 局限

1. **过程型评估的解空间不完备**：过程型评估假设人类专家已枚举所有有效解题路径，但未来模型可能发现或发明新定理，开辟替代路径，这会使现有标签失效。
2. **干预型评估的因果归因不精确**：总效应仅提供因果归因的下界，无法区分干预抑制基线行为与干预导致额外行为的情形。虽然最小准则 TPR 缓解了这一问题，但评估仍存在保守性。
3. **监控器本身的可靠性**：本文假设监控器（通常也是语言模型）的输出是可信的，但监控器自身也可能出错或产生幻觉。监控器退化评估部分缓解了这一问题，但并未完全解决。
4. **评估范围的代表性**：尽管评估套件涵盖多种行为类型和领域，但所有评估都基于可构造的测试环境，可能与真实部署中的不对齐行为分布不同。
5. **模型有机体的简单性**：本文训练的混淆模型是有概念验证性质的简单有机体，真实世界中模型可能演化出更复杂的混淆策略，超出当前评估的检测能力。
6. **外部模型评估受限**：由于 API 限制，外部模型仅在不使用工具的评估子集上测试，可能低估了它们在工具使用环境中的可监控性。

## 我的笔记

这篇论文在 AI 安全领域具有重要的方法论意义。它将"可监控性"从一个模糊概念转化为可量化、可比较的指标，并提供了系统性的评估工具包。g-mean² 指标的设计尤其巧妙，它直接解决了类别不平衡下标准指标失效的问题，使得跨模型比较变得有意义。

从工程角度看，论文中关于"可监控性税"的发现对实际部署有直接影响：安全团队不应仅根据能力选择模型，而需要将可监控性纳入模型选择的多目标优化中。有时使用更小但推理更充分的模型可能是更安全的策略，尽管成本更高。

过程型评估的思路——通过构造有限解空间、难猜测答案、易验证答案的问题来间接推断推理过程——是一个很有创意的妥协方案。它在无法直接访问模型内部状态的情况下，尽可能从外部行为反推内部机制。这一思路或许可以扩展到代码生成、科学推理等其他需要验证推理路径的领域。

后续问题提升可监控性的方法特别值得进一步探索。如果简单追问就能让模型暴露更多推理细节，那么设计更好的追问策略（例如针对性而非通用性追问）可能会带来更大收益。这与主动学习中的查询策略设计有相似之处。

一个潜在的后续研究方向是：将可监控性评估与 red-teaming 结合，不仅测量当前模型的可监控性，还主动搜索那些同时具有高不对齐风险和低可监控性的"危险区域"。这有助于在安全评估中优先关注最难以监控的最坏情况。
