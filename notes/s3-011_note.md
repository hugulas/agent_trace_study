## 原文摘要翻译

随着大语言模型智能体的不断演进，token 已经成为智能体化人工智能的核心经济原语。然而，token 的指数级消耗带来了严重的计算、协作与安全瓶颈。现有的综述文献仍然分散在系统优化、架构设计与可信性等孤立的技术领域中，缺乏一个统一的框架来评估输出质量与经济成本之间的根本权衡。为填补这一空白，本文首次对 token 经济学进行了全面综述。通过融合计算机科学与经济学，本文将 token 概念化为生产要素、交换媒介与记账单位，并在四维分类体系下综合现有文献：（1）微观层面（单智能体）：基于新古典厂商理论，在预算约束下优化要素替代；（2）中观层面（多智能体系统）：运用交易成本理论与委托-代理理论，最小化协作摩擦；（3）宏观层面（智能体生态系统）：通过机制设计解决拥堵外部性与定价问题；（4）安全层面：将对抗性威胁内化为内生经济约束。最后，本文展望了前沿方向，包括可微分 token 预算与动态市场，为可扩展的下一代智能体系统奠定理论基础。

## 创新点

1. **首次提出 Token Economics 的统一理论框架**。现有综述分散在系统优化、架构设计、可信性三个孤立阵营中，本文首次将 token 定义为兼具生产要素、交换媒介与记账单位三重经济属性的核心原语，建立了一个横跨微观、中观、宏观与安全四个维度的统一分类体系。

2. **建立计算机系统与经济学理论之间的同构映射**。本文将单智能体系统映射为新古典厂商理论中的独资企业，将多智能体系统映射为科斯交易成本理论下的层级组织，将开放智能体生态系统映射为多边平台市场，使系统优化问题获得了严格的经济学解释。

3. **将安全威胁内化为内生经济约束**。不同于传统安全综述将攻击与防御视为孤立技术问题，本文将对抗性威胁重新定义为经济成本模型中的约束项，使安全预算可以直接纳入总成本最小化框架。

4. **提出基于生产函数的系统化成本建模方法**。本文引入 CES 生产函数与全要素生产率（TFP）概念，将注意力优化、KV Cache 优化、量化剪枝、提示压缩等技术杠杆统一解释为对资本成本、要素替代弹性或全要素生产率的调控手段。

## 一句话总结

本文首次构建了 Token Economics 的统一理论框架，通过将单智能体、多智能体与开放生态系统分别映射为新古典厂商理论、交易成本理论与机制设计理论，把系统优化问题转化为可在统一成本-质量约束下求解的经济学问题。

## 研究问题

当前 LLM 智能体的 token 消耗正在呈指数级增长：OpenRouter 平台的周度 token 处理量从 2024 年 12 月的 0.4 万亿飙升至 2026 年 3 月的 27.0 万亿，增长约 67.5 倍。这种爆发式增长带来了三个层面的瓶颈：

- **计算瓶颈**：单次复杂任务需要反复反思、检索与多智能体同步，推理 token 急剧膨胀；
- **协作瓶颈**：多智能体系统中的通信、同步与冲突消解消耗大量通信 token；
- **安全瓶颈**：对抗性攻击在 token 生命周期的每个阶段都可能造成经济损失。

然而，现有综述文献严重碎片化。大多数前期综述可分为三大阵营：智能体架构综述、系统优化综述与可信性综述，但没有任何一部工作能够同时回答"在固定预算下如何获得最优输出质量"这一核心问题。本文的研究动机正是要填补这一空白，建立一个统一的双重视角框架，使系统设计与经济分析能够相互解释、相互指导。

## 数据与任务定义

本文是一篇综述性论文，其"数据"主要表现为对现有文献、基准测试与产业统计的综合梳理：

**产业统计**：以 OpenRouter 平台的周度 token 处理量作为 token 需求爆发式增长的实证依据（0.4T → 27.0T）。

**Token 经济分类**（Table 2）：论文将智能体生态系统中的 token 按其在推理生命周期中的生成、交换与消耗方式，划分为以下类别并赋予其经济属性：

- **输入 token（Input Token）**：作为原材料投入进入生产流程；
- **推理 token（Reasoning Token）**：作为内部加工过程中的中间产出，对应厂商的内部生产要素；
- **通信 token（Communication Token）**：在多智能体系统中承担交换媒介职能，对应交易成本；
- **外部 token（External Token）**：通过工具调用或检索从外部获取，对应外部采购的中间品。

## 方法主线

### 机制流程

本文的方法主线可以概括为四个递进层级，每一层都对应一种经济学理论映射：

1. **理论基础层：生产函数与成本模型**。输入为原始任务请求与各类 token 消耗量（$K$ 为物理计算资本，$M$ 为材料投入，$L$ 为推理步数/劳动），经过常替代弹性生产函数变换，输出为智能体任务输出质量 $Y$。总成本由资本租金、token 采购价格与推理时间成本加权求和得到：
   $$Y = A \cdot [\delta K^{\rho} + (1-\delta)M^{\rho}]^{\frac{\theta}{\rho}} \cdot L^{\beta} \cdot e^{\epsilon}$$
   $$TC = P_k \cdot K + P_m \cdot M + w \cdot L$$
   其中 $A$ 为全要素生产率，$\rho$ 控制要素替代弹性。
$\delta$ 为资本与材料投入的分配参数，$\theta$ 为规模报酬参数。
$\beta$ 为劳动弹性。

2. **微观层：单智能体资源路由**。输入为具体任务请求与可用预算上限，智能体在内部推理 token（$M_{int}$）与外部工具/检索 token（$M_{ext}$）之间进行要素替代，通过求解等成本线与质量约束的切点 $E^*$ 确定最优策略，输出为资源分配比例与最终答案。

3. **中观层：多智能体系统组织边界优化**。输入为任务复杂度、智能体拓扑结构与智能体数量 $N$，系统在节点生产成本（随 $N$ 增加而因专业化下降）与内部交易成本（随 $N$ 增加而因通信摩擦超线性上升）之间权衡，输出为总成本最小时的科斯最优组织规模 $N^*$。

4. **宏观与安全层：生态系统定价与威胁内化**。输入为市场供需曲线、系统拥堵状态与完整安全攻击面，通过机制设计（如动态定价、拍卖、监管政策）将外部性内部化，将对抗性威胁建模为安全约束下的成本增量，输出为市场均衡价格、最优拥堵税率与安全预算分配方案。

### 技术杠杆的经济学解释

论文将大量现有技术重新解释为对生产函数或成本模型参数的调控：

- **降低资本成本 $P_k$**：注意力优化、KV Cache 优化、量化（OPTQ）、剪枝（Movement Pruning）等；
- **提升全要素生产率 $A$**：混合专家模型、投机解码等；
- **减少内部材料投入 $M_{int}$**：提示压缩、文本凝练、潜空间推理、早退等；
- **降低外部 token 影子价格 $\tilde{P}_{ext}$**：工具选择与调用优化（MCP）、按需检索、结构化检索等；
- **积累知识资本**：Generative Agents 的记忆流、Reflexion 的错误反思、自组织记忆图等，通过 Arrow 学习曲线降低未来平均成本。

## 关键结果

由于本文是综述性质，"结果"主要表现为对已有技术方案的系统性归类与经济映射，而非新实验数据。以下是论文梳理出的核心发现：

1. **单智能体优化存在两条范式**：
   - **范式 A（工程优化）**：直接降低单位成本或提升单位产出效率，如注意力优化、量化、混合专家模型；
   - **范式 B（资源配置）**：在预算约束下动态调整要素组合，使系统趋近帕累托前沿 $E^*$，如提示压缩、工具调用路由、预算感知搜索。

2. **多智能体系统存在科斯最优规模 $N^*$**：随着智能体数量增加，节点生产成本因专业化而下降，但内部交易成本因通信摩擦而超线性上升，总成本曲线呈 U 型，最低点即为最优组织边界。

3. **记忆架构的三层资本结构**：
   - **工作记忆（流动资金）**：提示压缩、虚拟分页，降低当期可变投入；
   - **情景记忆（研发投资）**：错误反思与高层认知提取，沿 Arrow 学习曲线降低未来平均成本；
   - **持久结构化记忆（固定资产）**：自组织记忆图、持久记忆中心架构，将一次性上下文消耗转化为可复用的知识资本。

## 深度分析

**经济学隐喻的技术有效性**。本文最有价值的贡献在于建立了一套系统性的"经济语法"来重新描述智能体优化问题。例如，将提示压缩解释为"将重复性 token 支出摊销为一次性固定投资"，将工具调用解释为"在内部参数推理与外部检索之间的要素替代"，这些隐喻不仅具有解释力，还能指导新的算法设计。当研究者面对一个新的成本-质量权衡问题时，可以直接借用厂商理论中的等成本线、生产前沿等概念进行直觉推理。

**生产函数参数的实操含义**。常替代弹性生产函数中的参数具有明确的工程对应：$\rho$ 越接近 1，意味着内部推理与外部检索之间的替代弹性越高，系统越灵活；$\theta > 1$ 表示规模报酬递增，对应大模型涌现能力；$A$ 的提升对应架构创新（如混合专家模型、投机解码）。然而，论文并未给出这些参数在具体模型上的估计方法或实证数值，这使得生产函数目前仍停留在概念模型层面。

**安全层面的经济化重构**。将对抗性威胁内化为经济约束是一个新颖的视角。论文提出，安全成本不应被视为独立预算，而应当作为 $TC$ 中的一个附加项：
$$TC_{security} = TC_{base} + C_{defense} + P_{attack} \cdot L_{attack}$$
其中 $P_{attack}$ 为攻击发生概率，$L_{attack}$ 为攻击造成的预期损失。这种建模方式使得安全投资可以与性能优化在同一框架下比较边际收益。

**综述方法的局限**。论文对大量技术进行了高度概括的经济映射，但某些映射可能过于简化。例如，混合专家模型的"专业化收益"与经济学中的劳动分工之间存在重要差异（其路由机制本身也消耗 token，这一点在生产函数中未被显式建模）；投机解码的加速效果依赖于草稿模型与目标模型的分布匹配度，这种结构性依赖关系难以用单一全要素生产率提升来概括。

## 局限

1. **缺乏原创实验与定量验证**。作为综述论文，本文未报告任何新的实验结果。所有经济映射均为理论构建，缺乏在真实系统上的参数标定或因果验证。例如，常替代弹性生产函数中的 $\rho$、$\delta$、$\theta$ 等参数在具体模型上的取值仍然未知。

2. **经济隐喻的适用边界未被严格讨论**。并非所有系统优化问题都能被干净地映射为经济问题。例如，多智能体系统中的通信开销不仅取决于 token 数量，还取决于拓扑结构、同步协议与失败恢复机制，这些因素在简单的交易成本模型中可能被过度简化。

3. **对安全威胁的经济建模仍较初步**。虽然提出了将安全内化为成本约束的方向，但论文未给出攻击概率 $P_{attack}$ 或损失函数 $L_{attack}$ 的具体估计方法，也未讨论不同攻击类型（提示注入、数据投毒、模型窃取等）在经济成本结构中的差异。

4. **动态市场与可微分预算的方向仍停留在展望**。论文在结尾提出"可微分 token 预算"与"动态市场"作为前沿方向，但未展开技术实现路径。这些概念如何与现有的自动微分框架（如 PyTorch）或强化学习训练流程结合，仍需后续工作填补。

## 我的笔记

本文的价值不仅在于综述，更在于提供了一种跨学科的共同语言。在实际的智能体基础设施开发中，团队往往面临一个困境：系统工程师关注延迟与吞吐，产品经理关注成本，研究员关注效果，但三方缺乏统一的决策框架。Token Economics 提供了一个天然的语言——"在给定质量约束下最小化总成本"——使不同角色可以在同一目标函数下讨论权衡。

从工程视角看，本文的"范式 B（资源配置）"对当前智能体框架的设计最具启发。现有的推理-行动循环、自我反思、思维树等框架大多默认使用固定策略（如固定搜索深度、固定反思轮数），而本文暗示，更优的做法是将这些超参数视为可在运行时根据预算动态调整的控制变量。这与近期出现的预算感知推理研究趋势高度一致。

一个值得深入思考的问题是：如果 token 真的成为智能体经济系统的"货币"，那么是否存在"通货膨胀"（模型提供商集体涨价）、"汇率波动"（不同模型间 token 价格的相对变化）与"资本管制"（API 限流、区域封锁）等宏观现象？论文的宏观层面分析仅涉及拥堵定价与寡头竞争，对更复杂的金融化现象（如 token 期货、算力衍生品）尚未触及，这可能是未来研究的重要方向。
