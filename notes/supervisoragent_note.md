## 原文摘要翻译

尽管多智能体系统在复杂任务中表现优异，但随着运行复杂度的提升，其自主性的增长往往导致严重的效率低下，例如过度的令牌消耗以及由错误信息引发的失败。现有方法主要聚焦于事后失败归因，缺乏能够主动、实时提升系统鲁棒性与效率的干预手段。为此，本文提出 SupervisorAgent，一种轻量且模块化的运行时自适应监督框架，该框架无需修改底层智能体的架构即可工作。在无需大语言模型参与的自适应过滤器触发下，SupervisorAgent 在关键节点进行干预，主动纠正错误、引导低效行为并净化观察信息。在具有挑战性的 GAIA 基准测试上，SupervisorAgent 将 Smolagent 框架的令牌消耗平均降低 29.68%，同时不损害其成功率。在五个额外基准测试（数学推理、代码生成和问答）以及多种最先进基础模型上的大量实验，验证了本文方法的广泛适用性与鲁棒性。

## 创新点

1. **运行时自适应监督范式**。现有方法多聚焦于事后归因，而本文首次系统性地将实时监督引入多智能体系统的运行流程，在错误传播发生前进行主动拦截，从而同时提升鲁棒性与经济性。
2. **无需大语言模型的自适应过滤机制**。为了在不引入高额计算开销的前提下决定何时干预，本文设计了一套基于启发式规则的轻量过滤器，避免了为每次判断都调用大语言模型，这是实现整体降本增效的关键设计。
3. **三类干预策略的协同设计**。本文将监督动作细分为错误主动纠正、低效行为引导和观察信息净化三种策略，并针对不同风险类型匹配不同强度的干预，形成了从简单批准到深度验证的连续干预谱系。
4. **非侵入式模块化架构**。SupervisorAgent 以元智能体的形式存在，无需修改底层智能体的架构或训练流程，可直接叠加在 Smolagent、OAgents 等现有框架之上，具备良好的即插即用特性。

## 一句话总结

SupervisorAgent 通过一套无需大语言模型的自适应过滤器和三类运行时干预策略，在不修改底层智能体架构的前提下，将多智能体系统的令牌消耗平均降低约 30%，同时保持任务成功率。

## 研究问题

本文解决的核心矛盾可概括为：多智能体系统的能力与复杂度持续提升，但其运行鲁棒性与经济性却在下降。具体而言，本文识别出两大效率瓶颈来源：

- **观察信息过载**：智能体在处理冗长的网页内容或工具输出时，上下文窗口被大量无关信息淹没，不仅推高了令牌成本，还可能掩盖关键信息，导致智能体偏离任务目标。
- **策略性低效**：智能体可能陷入重复的行动循环，或选择过于复杂的求解路径，造成计算资源的浪费。

此外，现代多智能体系统中漫长的交互链为错误传播提供了温床。单个智能体产生的幻觉信息一旦写入记忆，便会污染后续所有下游智能体的推理过程。现有工作主要关注事后失败归因，无法在运行过程中实时打断这一错误传播链。本文的研究动机正是在于填补这一空白：如何在运行时对多智能体系统进行轻量、自适应的监督，从而在不牺牲成功率的前提下显著降低令牌消耗并提升鲁棒性。

## 数据与任务定义

本文在六个基准测试上展开评估，覆盖数学推理、代码生成、问答和通用问题解决四个领域：

| 基准测试 | 领域 | 样本量 | 说明 |
|---------|------|--------|------|
| GAIA validation set | 通用问题解决 | 完整验证集 | 主要基准，分 Level 1/2/3 |
| AIME 2024 | 数学推理 | 完整数据集 | 竞赛级数学问题 |
| GSM8k-Hard | 数学推理 | 随机 600 条 | 更难子集 |
| HumanEval | 代码生成 | 完整数据集 | 函数级代码生成 |
| MBPP | 代码生成 | 完整数据集 | 入门级编程问题 |
| DROP | 问答 | 800 条子集 | 离散推理问答 |

评估指标包括：
- **成功率**：pass@k，即前 k 个候选中至少有一个正确的比例。
- **令牌成本**：每个任务消耗的令牌总量（输入加输出）。
- **延迟**：任务完成所需时间（秒）。

基线分为两类：
- 单智能体：vanilla LLM、Self-Consistency CoT（3 answers）、CodeAgent。
- 多智能体：Smolagent、OAgents、MetaAgent、OWL（角色扮演）、AWorld。

主要实验以 Smolagent 框架为测试平台，基础模型主要采用 GPT-4.1，并在 GAIA 上额外测试了 Gemini-2.5-pro-0605 和 Qwen3-235B-2507 以验证模型无关性。

## 方法主线

### 形式化定义

本文将受监督的多智能体系统（Supervised Multi-Agent System, SMAS）形式化如下。对于每个智能体，定义其工作单元为：

$$W = (N, Q_g, Q_l, T_l, S)$$

其中 $N$ 为被监督智能体的名称，$Q_g$ 和 $Q_l$ 分别为全局与局部查询，$T_l$ 为局部轨迹，$S$ 为当前状态。扩展工作单元进一步纳入全局交互轨迹：

$$W_{\text{ext}} = W \cup \{T_g\}$$

其中 $T_g$ 记录所有智能体交互的全局轨迹。

### 监督目标：三大高风险交互点

SupervisorAgent 的监督目标聚焦于三类高风险交互通道：

1. **Agent-Agent 交互**：智能体之间的通信是错误信息传播的主要途径。
2. **Agent-Tool 交互**：智能体调用外部工具时，工具返回的错误或冗长输出可能引发后续连锁问题。
3. **Agent-Memory 交互**：从短期或长期记忆中检索信息时，可能获取到过时或错误的记忆条目。

这三类交互点是系统中错误与低效被引入并扩散的核心渠道。

### 自适应过滤：何时监督

若对每一次交互都进行监督，其计算开销将极为高昂，反而违背提升效率的初衷。因此，本文的核心设计之一是一个**轻量的、无需大语言模型的自适应过滤器**。该过滤器基于启发式规则对当前执行历史、观察内容和错误状态等信号进行快速评估，仅在关键节点触发监督。过滤逻辑的核心超参数针对不同基准进行了差异化配置，详见附录。

### 干预策略：如何监督

当过滤器触发后，SupervisorAgent 首先诊断当前交互的风险类型，所有风险构成集合：

$$\mathcal{C} = \{c_{\text{error}}, c_{\text{inefficient}}, c_{\text{excessive}}\}$$

三类风险分别激活以下核心策略：

- **错误主动纠正**（$c_{\text{error}}$）：在错误被提交记忆或传播给下游前进行拦截与纠正。
- **低效行为引导**（$c_{\text{inefficient}}$）：当智能体陷入重复循环或低效路径时，提供策略性指引以精简求解步骤。
- **观察信息净化**（$c_{\text{excessive}}$）：对冗长观察进行压缩与净化，保留关键信息的同时降低上下文负载。

对应地，干预动作 $a$ 的取值包括：

$$
a = \begin{cases}
\{\text{approve}, \text{provide\_guidance}\} & \text{if } c = c_{\text{inefficient}} \\
\{\text{correct\_observation}\} & \text{if } c = c_{\text{excessive}}
\end{cases}
$$

对于 $c_{\text{error}}$，系统则执行错误修正流程。

### 机制流程

SupervisorAgent 的完整监督工作流可分解为以下四个步骤：

1. **输入：交互事件与扩展工作单元**。系统接收当前智能体产生的交互事件，事件类型为 Agent-Agent、Agent-Tool 或 Agent-Memory 三者之一。同时，系统读取该智能体的扩展工作单元 $W_{\text{ext}}$，其中包含局部查询 $Q_l$、局部轨迹 $T_l$、全局查询 $Q_g$ 以及全局交互轨迹 $T_g$，这些信息共同构成监督决策所需的完整上下文输入。

2. **动作：自适应过滤判断**。过滤器依据 Algorithm 1 中的启发式规则，对三项信号进行快速评估：当前执行步数是否达到检查间隔阈值、观察长度是否超过字符上限、以及当前是否存在工具执行错误。若三项信号均未超限，过滤器直接输出 **approve** 信号，本轮监督结束；若任一信号超限，则将交互事件标记为高风险并送入下游的监督模块。

3. **动作：风险诊断与干预选择**。监督模块将 $W_{\text{ext}}$ 注入上下文窗口，结合任务目标与历史轨迹进行风险诊断，将交互归类为错误型、低效型或冗余型。随后从干预谱系中选择对应动作：对冗余型执行 **correct_observation** 以压缩观察；对低效型执行 **provide_guidance** 以给出精简策略；对错误型执行 **error correction** 以阻断错误传播。

4. **输出：修正后的交互或批准信号**。监督模块输出经过处理后的交互内容（如净化后的观察文本或纠正后的动作指令），或直接输出批准信号。该输出作为下一时刻被监督智能体的输入，从而完成一次监督闭环，系统继续处理后续交互事件。

## 关键结果

### GAIA 主实验

在 GAIA 验证集上，以 Smolagent 为基线的实验结果显示，SupervisorAgent（SMAS）在保持竞争力的 pass@k 成功率的同时，显著降低了平均令牌成本。具体而言：

- pass@1 准确率从基线的 34.62% 提升至 **38.46%**（提升 3.84%）。
- 平均令牌消耗从基线的 611.90K 降至 **427.72K**（降低 30.10%）。
- 综合三个难度层级，SMAS 平均降低令牌消耗 **29.68%**。

在 GAIA Level 2 的任务上，SMAS 不仅将令牌成本降低 35%，还将消耗的方差降低 **63%**，表明系统在降低均值成本的同时显著提升了运行稳定性。

在其余五个基准测试上，SupervisorAgent 普遍实现了帕累托改进：

### 成本效益分析

自适应过滤器的设计体现了明确的成本意识。若取消过滤器、对所有交互进行监督，大语言模型的调用次数将大幅上升，监督本身的成本可能吞噬掉干预带来的节省。过滤器通过启发式规则将监督调用限制在真正的高风险节点，使得整体系统实现净收益。不过，过滤器的阈值需要针对不同框架和基准进行差异化调参（见附录表 5），这引入了一定的工程适配成本。

## 局限

1. **单点监督瓶颈**：SupervisorAgent 作为中心化的元智能体，其自身的推理能力和上下文窗口成为系统瓶颈。在 OAgents 框架中，单次观察长度曾超过 20 万字符，此时 SupervisorAgent 在压缩观察时存在幻觉或遗漏关键细节的风险。

2. **过滤器阈值需要手动调参**：虽然自适应过滤器避免了 LLM 调用，但其启发式阈值（步数间隔、长度阈值等）针对不同基准和框架需要差异化配置，尚未实现完全自适应的参数学习。

3. **监督本身的延迟开销**：尽管令牌成本显著降低，但监督模块的引入增加了额外的推理延迟。表 7 的消融数据显示，完整框架的平均延迟高于无监督基线，在对延迟极端敏感的场景中这可能构成限制。

4. **未覆盖的外部工具成本**：本文的成本分析主要聚焦于 LLM 令牌消耗，未全面计入外部工具 API 的调用成本。对于那些大量依赖付费外部工具（如搜索引擎、代码执行环境）的任务，令牌节省在总成本中的占比可能被稀释。

5. **成功率上限仍受基础模型能力约束**：SupervisorAgent 的干预只能纠正运行层面的错误与低效，无法突破底层基础模型本身的能力上限。对于超出基础模型知识范围的任务，监督机制难以带来质的提升。

## 我的笔记

SupervisorAgent 与同期工作 GuardAgent 形成有趣的互补关系：GuardAgent 聚焦于准确率保障，通过拦截错误动作防止任务失败；而 SupervisorAgent 的目标更为宽泛，追求系统整体的效率与鲁棒性，其干预谱系涵盖错误纠正、效率指引和观察净化。两者的差异本质上反映了多智能体系统研究从“保证做对”到“做得又快又省”的演进趋势。

从更宏观的视角看，本文提出的运行时监督范式对当前智能体系统的工程实践具有直接参考价值。随着 Operator、DeepResearch 等产品将多智能体系统推向真实用户场景，令牌经济和延迟开销不再是纯粹的学术指标，而是直接决定商业模式的可行性因素。SupervisorAgent 的非侵入式设计使其可以在现有框架上低成本部署，这对于工业界而言尤为重要。

一个值得后续探索的方向是：能否将自适应过滤器的启发式阈值也纳入学习范畴，让监督系统根据在线反馈自动调整触发策略，从而进一步降低人工调参成本。此外，当系统规模扩展到数十甚至上百个智能体时，中心化监督的瓶颈将更加突出，分布式或层级化的监督架构可能是必然的演进方向。
