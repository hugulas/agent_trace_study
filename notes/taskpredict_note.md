# Agent psychometrics: 智能体代码基准的任务级性能预测

## 核心信息

- 标题: Agent psychometrics: Task-level performance prediction in agentic coding benchmarks
- 作者: Chris Ge, Daria Kryvosheieva, Daniel Fried, Uzay Girit, Kaivalya Hariharan
- 年份: 2026
- 会议/期刊: ICLR 2026 Workshop on Agents in the Wild
- DOI: 10.48550/arXiv.2604.00594
- 论文链接: https://arxiv.org/abs/2604.00594

## 原文摘要翻译

随着基于大语言模型的代码生成从静态单步生成转向与工具和环境进行多步智能体交互，理解哪些任务会对智能体构成挑战以及原因变得愈发困难。当前实践进一步加剧了这一问题：智能体性能通常以基准测试上的聚合通过率来衡量，但单一数字指标掩盖了基准内部任务的多样性。本文提出了一套面向智能体代码评测场景的个体任务成功或失败预测框架。我们的方法用从任务中提取的丰富特征来增强项目反应理论，这些特征包括问题陈述、仓库上下文、解决方案和测试用例，并引入了一种将智能体能力分解为大语言模型能力与脚手架能力的新参数化方式。这种参数化使我们能够聚合来自异构排行榜的评估数据，并准确预测未见基准上的任务级性能，以及未见过的大语言模型与脚手架组合的性能。我们的方法对基准设计者具有实用价值：他们可以在不运行计算成本高昂的智能体评估的情况下，更好地校准新任务的难度。

## 创新点

1. **将项目反应理论扩展到智能体代码评测领域**。传统项目反应理论用于教育心理测量中的考生与试题交互建模，本文将其适配到智能体代码基准测试，解决了聚合通过率无法反映任务异质性的问题。这使得基准设计者能在不实际运行昂贵智能体评估的前提下预估任务难度。

2. **双通道任务特征编码**。论文提出了两种互补的任务特征提取方式：基于开放权重大语言模型的嵌入向量，以及基于大语言模型作为评判者的多维度评分特征。两种特征均可从任务的静态产物中提取，包括问题陈述、仓库状态、测试用例和解决方案，无需执行智能体。

3. **智能体能力的可加性分解**。论文首次将智能体能力分解为独立的大语言模型能力参数与脚手架能力参数，且无需成对交互项。该分解的验证实验显示，固定脚手架上训练得到的智能体能力与分解后的大语言模型能力之间的皮尔逊相关系数高达 0.974。

4. **跨基准聚合与泛化**。通过上述参数化，论文实现了在四个异构代码基准上的联合训练与跨基准预测，证明了方法在分布外任务上的稳健性。

## 一句话总结

本文通过用任务静态特征增强项目反应理论并将智能体能力可加分解为大语言模型能力与脚手架能力，实现了对智能体代码基准中个体任务成功概率的高效预测与跨基准泛化。

## 研究问题

智能体代码评测正从静态单步生成转向多步交互式评估。这种转变带来三个核心挑战：

- **环境交互的复杂性**。测试用例往往检验问题陈述中未提及的代码库属性，或允许未充分指定的提交通过，导致任务存在多条有效解决路径和难以预料的边缘情况。

- **任务异质性**。同一基准内的不同任务可能因不同原因导致智能体失败，而当前常见的单一聚合通过率指标无法捕捉这种差异。

- **评估成本极高**。获取任务级理解需要在每个任务上运行多套智能体，例如某自改进软件智能体在 SWE-bench Verified 上的单次运行成本高达两万两千美元。

因此，本文的核心问题是：如何在不运行完整智能体评估的前提下，高效预测智能体在代码基准中每个个体任务上的成功或失败？

## 数据与任务定义

### 基准数据集

论文使用四个公开的智能体代码评测基准，每个数据集均提供智能体在每条任务上的详细成功或失败记录：

- **SWE-bench Verified**：真实 GitHub 议题的代码修复任务
- **SWE-bench Pro**：SWE-bench 的扩展版本
- **GSO**：面向开源软件工程的智能体评测
- **Terminal-Bench 2.0**：测试智能体在终端环境中利用工具调用迭代解决问题的能力

### 实验设置

论文在四种设置下评估响应预测性能：

1. **新任务**：在单个基准的随机任务子集上训练，预测该基准剩余任务上的性能，验证任务特征是否能解释任务难度。
2. **新响应**：预测已见过任务但由未见过智能体产生的响应，验证智能体特征分解的有效性。
3. **新基准**：在三个基准上训练，预测第四个未见基准上的性能，验证跨基准泛化能力。
4. **自适应任务选择**：利用预测的难度分数选择信息量最大的任务子集进行实际评估，以降低评估成本。

## 方法主线

### 核心思想

本文扩展标准项目反应理论，使其能够利用任务特征和智能体特征进行任务级性能预测，而不是将智能体和任务视为无结构的离散编号。

标准项目反应理论的 logistic 模型为：

$$P(y_{ij} = 1 \mid \theta_i, \beta_j) = \sigma(\theta_i - \beta_j)$$

其中 $\theta_i$ 是被试（此处为智能体）的能力参数，$\beta_j$ 是试题（此处为任务）的难度参数，$y_{ij}$ 是二元正确性指标。

### 任务特征编码

论文考虑两种将任务转换为特征向量的方法：

1. **嵌入特征**：将任务文本输入开放权重大语言模型，获取其隐藏层嵌入向量作为特征表示。

2. **评判者特征**：用大语言模型对任务按预设的多维度评分标准进行打分，例如任务结果是否易于验证、所需领域知识的深度、代码修改的复杂度等。

两种方法均可单独使用，也可拼接组合。论文在特征源消融实验中逐步加入问题陈述、仓库状态、测试用例和解决方案四类信息源。

### 响应预测方法

#### 项目反应理论与任务特征

借鉴前人工作，论文用一个线性模型从任务特征向量预测任务的项目反应理论难度参数。但与联合训练项目反应理论参数和线性模型权重的做法不同，本文采用分阶段训练策略。首先训练标准项目反应理论的能力参数和难度参数以最大化对数似然，然后冻结这些参数，再用岭回归拟合任务特征到冻结后的难度参数，最优正则化超参数通过训练集内五折交叉验证的最小均方误差选取。

实验表明，这种分阶段策略优于联合对数似然最大化。

#### 项目反应理论与智能体特征

论文将智能体能力参数化为大语言模型能力参数与脚手架能力参数之和：

$$P(y_{msj} = 1 \mid \theta_m, \theta_s, \beta_j) = \sigma(\theta_m + \theta_s - \beta_j)$$

这一可加分解是严格比标准项目反应理论表达能力更弱的约束，但实验表明其性能损失几乎可忽略：标准项目反应理论与带智能体特征的版本在 AUC-ROC 上的平均差异仅为 0.0005，平均绝对差异为 0.002。

### 机制流程

1. **输入：任务静态产物与智能体标识**
   输入信号包括任务的议题陈述、仓库上下文、参考解决方案、测试用例，以及智能体对应的大语言模型型号和脚手架名称。这些产物无需运行智能体即可获得。

2. **任务特征提取与难度预测**
   对每条任务，分别通过大语言模型嵌入和评判者评分提取特征向量。将特征输入预训练的岭回归模型，预测该任务的难度参数 $\hat{\beta}_j$。

3. **智能体能力参数查表**
   根据智能体的大语言模型型号和脚手架名称，从多基准联合训练得到的参数表中查表获取对应的大语言模型能力参数 $\theta_m$ 和脚手架能力参数 $\theta_s$。

4. **输出：成功概率估计**
   将估计的能力参数与难度参数代入扩展后的项目反应理论模型，输出该智能体在该任务上的成功概率 $P(y_{msj}=1) = \sigma(\theta_m + \theta_s - \hat{\beta}_j)$。该概率可用于基准设计时的难度校准，或用于自适应选择信息量最大的评估任务子集。

## 深度分析

### 可加分解的合理性与代价

将智能体能力严格约束为两参数之和且无成对交互项是一个强假设。作者明确指出，这一约束使模型严格不如标准项目反应理论灵活，但实验表明性能损失几乎可忽略。这种以极小的建模代价换取跨基准可比较性的权衡是本文的关键设计决策。

### 与联合训练的比较

论文在附录中对比了分阶段训练与联合对数似然最大化。结果显示分阶段策略在多数基准上的 AUC 更高。这暗示在项目反应理论参数空间中，难度参数与特征线性权重之间存在复杂的耦合，联合优化可能导致次优解。

## 局限

1. **因果性未确立**。论文无法证明智能体任务产物对难度存在因果影响。这些产物可能只是暴露问题陈述中已隐含的信息，而非本身生成难度。作者建议未来通过构建反事实任务来研究因果关系。

2. **可加性假设的未来风险**。当前实验支持大语言模型能力与脚手架能力独立可加，但作者警告：随着越来越多模型与其专用脚手架联合训练，这种独立性可能不再成立。

3. **智能体特征仍为类别型**。当前方法将大语言模型和脚手架视为离散类别特征，无法对完全未见过的新智能体进行特征提取和性能预测。未来需要找到易于获取的定量或语义层面的智能体特征。

4. **跨基准泛化存在性能衰减**。虽然跨基准预测击败了基线方法，但与同基准 Oracle 相比仍有明显差距。这意味着方法更适合作为新基准难度预估的初筛工具，而非完全替代实际评估。

## 我的笔记

### 与相关工作的联系

本文的方法与将项目反应理论用于自然语言处理评测的工作一脉相承，但将应用场景从静态问答任务扩展到多步智能体交互任务，这是一个实质性的领域迁移。任务特征编码中评判者特征的思路也与当前使用大语言模型进行自动评估的趋势一致，但本文的独特之处在于将评判者输出结构化后用于预测模型参数，而非直接打分。

### 对基准设计的启示

本文最直接的实用价值在于为基准设计者提供了不运行智能体即可预估任务难度的工具。在当前智能体评测成本持续攀升的背景下，这种预测能力可以显著降低新基准迭代的门槛。不过需要注意的是，本文的最佳预测器在 SWE-bench Verified 上达到 0.842 的 AUC，虽然远超基线方法的 0.718，但距离 Oracle 的 0.945 仍有约 0.1 的差距，说明预测器更适合用于初步筛选和校准，而非作为最终评估的替代。

### 对强化学习训练的可能应用

作者在讨论中提到，这些预测器可以指导强化学习 rollout 的任务选择。如果预测器能准确识别当前智能体最可能从中学到东西的任务子集，那么它不仅可以降低评估成本，还可以提升训练效率。这一方向值得进一步探索。

### 方法论层面的思考

将教育心理测量中的经典工具与现代大语言模型特征提取相结合，是本文方法论上的亮点。这种跨学科迁移的成功表明，在智能体评测这个新兴领域，传统统计方法经过适当扩展后仍然具有强大生命力。同时，智能体能力的可加分解也为未来更细粒度的能力归因分析提供了模板。
