| 论文 | 核心发现 | 与 Token Economics 的关联 |
|------|---------|--------------------------|
| **p-001** | Avg tokens / step：133 \| 169 \| 330；Avg tokens / trajectory：4,889 \| 6,415 \| 16,484。不同失败类型轨迹的 token 消耗差异可达 **3.4 倍**。 | 失败轨迹的长度本身直接对应经济成本，诊断越复杂的失败，消耗越多 token。 |
| **p-002** | ReAct 智能体平均 token 消耗 **16,941**，但准确率仍低于 Flash。GPT-3.5-w-Shell 超过 5 步后准确率不再提升，**仅增加令牌消耗**。 | "自我修复快速饱和"现象意味着追加推理步骤的边际收益为零，但边际成本持续为正。 |
| **p-004** | 平均每条轨迹评估成本：Total tokens 约 **77,621**（输入 69,000 + 输出 8,500），Estimated cost **$0.019**。 | 过程规范验证本身就需要大量 token，安全审计的经济成本必须计入总预算。 |
| **p-005** | PROBE 旁路设计的提示词开销：平均增加 **1.07K**（SWE-bench）、**0.45K**（EnterpriseOps-Gym）、**0.39K**（AIOpsLab）；反馈生成延迟中位数 **210.43 秒**。 | 诊断层附加到现有工作流时，其额外 token 和延迟开销必须在预算约束下核算。 |
| **p-006** | 追问模型并收集后续思维链，可**在不增加智能体计算成本的情况下**提升监控效果；但更大模型虽推理 effort 更高、可监控性更好，**总体推理计算成本也更高**。 | 存在明确的"可监控性税"：更安全的选择往往意味着更高的 token 成本。 |
| **p-007** | 更高推理强度让思维链更长、更容易被监控，但**增加推理成本**。生产系统需在特定任务上主动决定是否支付"可监控性成本"。 | 直接印证了安全/监控与经济性之间的显性权衡。 |
| **p-009** | 在长程设置中，早期工具错误会改变后续整个轨迹，**增加 token 消耗**并带来下游风险。隐藏状态对最后 32 个预行动 token 进行均值池化，以平衡信号质量与计算开销。 | 长程任务的早期错误具有成本放大效应，每个决策点的计算开销都需要精细控制。 |
| **p-010** | Continual Harness 从零开始达到 100% 里程碑，中位成本仅 **$130**；最简基线为 $215，**成本降低约 40%** 且无完成度损失。 | 在线自适应可显著降低 harness 工程的经济门槛，减少人工设计的沉没成本。 |
| **p-011** | AHE 将整体标记消耗降低 **32%**（相对 ACE）、**21%**（相对 TF-GRPO）和 **12%**（相对种子）；将行为编码到工具/中间件/记忆中，避免了仅调提示基线每轮重新推导的成本。 | 架构层面的外部化（记忆+工具）比提示工程更具规模经济性。 |
| **p-012** | 记忆检索、技能加载、协议模式、工具描述与模型推理轨迹对**有限 token 预算**竞争；扩展任一模块的上下文占用必然压缩其他模块空间。 | 单智能体内部存在"预算竞争"，Harness 必须动态协调相对预算分配。 |
| **p-013** | Meerkat 在大型仓库上的推理 token 消耗显著高于逐轨迹监控器；**13 倍 token 增长**才能带来 **5 倍检测概率提升**。 | 跨轨迹安全检测的边际成本极高，需要显式的资源-收益权衡决策。 |
| **p-014** | AgentTrace 通过 Python 反射实现运行时零侵入插桩，**兼顾低延迟与低开销**；建议引入分层存储——热数据高细粒度、温数据压缩指标、冷数据归档至低成本对象存储。 | 可观测性基础设施本身也消耗资源，需要按成本敏感度分级管理。 |
| **p-015** | 仅将输入格式从 JSONL 替换为 VCC 编译后的视图，**token 消耗削减一半到三分之二**；智能体轨迹中 **40%~60%** 的 token 被识别为无用、冗余或过期信息，移除不会降低性能。 | 上下文格式工程是最直接的"免费省钱"策略，大量 token 被无意义的数据格式浪费。 |
| **p-016** | 早期子规划错误具有路径依赖性，一旦选择错误分解方式，后续动作空间被持续约束，**回滚成本指数级增长**。 | 长程任务中早期决策失误的纠正成本呈非线性膨胀，预算分配应向前期规划倾斜。 |
| **p-017** | 混合协议对执行合理性和扰动稳定性的开放式判断需要 **GPT-5.4** 作为评判模型，**评估成本显著高于纯规则基准**。 | 安全审计的评估基础设施本身昂贵，需要纳入总拥有成本（TCO）。 |
| **p-018** | EAGER 检测延迟均控制在约 **5 秒** 以内；区别于现有方法依赖大型评判语言模型对每条轨迹独立进行**高成本推理**，EAGER 通过专用表示模型将故障检测转化为向量近邻检索，**大幅降低单次检测开销**。 | 从通用 LLM 推理到专用轻量模型的范式转移，是降低运维 token 成本的关键路径。 |
| **p-019** | 传统运维解决一起事件需 3 名资深工程师协作约 2.5 小时；系统将其缩短至 **126 秒**。使用 **14B 开源模型**作为推理核心，避免向第三方 API 发送敏感数据，**将推理成本控制在企业可承受范围内**。 | 开源小模型在特定垂直场景下可替代闭源大模型，实现成本与隐私的双重优化。 |
| **p-020** | SeqCV 平均运行时间 **217 秒**，相比 Flow 实现 **2.5 倍加速**，相比 Atom 实现 **2.1 倍加速**；在计算量最大的六个任务上，运行时间削减 **51%~82%**。选择函数仅输出决策而非长文本，**显著降低计算开销**。 | 工作流验证机制的设计直接影响执行效率，简洁的决策接口可减少冗余 token 生成。 |
| **p-021** | MAST 明确承认："效率（如冗余对话导致的 token 浪费）、**成本、延迟**、鲁棒性与安全性等维度同样关键，但这些**未被纳入当前分类法**。" 扎根理论分析阶段**每位专家需投入 20 小时以上**。 | 学术界的多智能体失败分析框架系统性忽视了经济维度；人工标注成本极高。 |
| **p-022** | 混合归因方法 token 消耗是全局法的约 **8.7 倍**；三位标注者分别花费 **30.9、30.2、23.2 小时**完成标注。 | 高精度故障归因的直接代价是 token 成本的近 9 倍增长，且人工标注极度耗时。 |
| **p-023** | 4 智能体时负载均衡性从 0.89 降至 **0.74**；EvoCF 每步额外引入三个模块，**每步延迟与令牌消耗仍高于 LLaMAR 基线**。 | 多智能体扩展存在"协调税"，智能体数量增加时通信与计算的边际成本上升。 |
| **p-024** | NSI 在单技能调用中封装约 **7.4 个原子步骤**，平均原子步骤数提升 **100%~140%**；但技能归纳涉及多次 LLM 调用，在资源受限场景下可能难以实时执行。 | 神经符号抽象可压缩推理步骤，但归纳阶段的 LLM 调用成本不可忽视。 |
| **p-025** | AgentSight 平均运行时开销仅为 **2.9%**（仓库编译 0.4%，代码生成 4.9%）；清晰的角色划分可**缩短总运行时间与 token 成本**。 | 系统级可观测性可以极低开销实现，且组织架构优化本身就能降低 token 消耗。 |
| **p-026** | MCTS 模拟次数从 1 增加到 3 时准确率提升 7 个百分点；从 3 增加到 5 仅提升 2 个百分点。**m=3 是计算效率与性能的良好平衡点**。全参数微调奖励模型带来**较高的训练成本**。 | 模拟次数存在收益递减点，超过该点的额外计算是经济浪费；训练成本可通过 LoRA 等参数高效微调降低。 |
| **p-027** | 测试时扩展（顺序扩展）带来的提升远超过语义信息表示带来的差异；**计算预算的重新分配（从训练转向推理）可能比训练信号的精心设计更有效**。 | 对于代码生成类任务，推理时的计算投入比训练阶段的信号工程更具成本效益。 |
| **p-028** | FLARE 相比单次思维链增加了计划、代码、搜索、答案**四次顺序生成**，**推理延迟与 token 消耗显著更高**。 | 忠实度提升以多轮生成为代价，每次额外生成都直接转化为可量化的经济成本。 |
| **p-029** | LLM judge 本身存在**成本与延迟问题**；建议微调小参数分类器（如 **1B-3B**）替代 LLM judge，降低评估成本并提升稳定性。 | 评估基础设施的降本路径：用大模型标注数据，用小模型执行高频评估。 |
| **p-030** | 失败动作感知的目标函数**不依赖额外的模型参数，计算开销极小**；最大序列长度 **1,000 个 token**，超过时截断最旧的动作-观察对。 | 极简的文本规则匹配即可实现有效的策略优化，无需昂贵的模型训练。 |
