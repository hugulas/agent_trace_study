# MASPrism: Lightweight Failure Attribution for Multi-Agent Systems Using Prefill-Stage Signals

## 核心信息

- **标题**: MASPrism: Lightweight Failure Attribution for Multi-Agent Systems Using Prefill-Stage Signals
- **作者**: Yang P. Liu, Hongjiang Feng, Junsong Pu, Zhuangbin Chen
- **年份**: 2026
- **来源**: ArXiv.org
- **DOI**: 10.48550/arXiv.2605.07509
- **arXiv**: 2605.07509
- **PDF**: `[c-006]-masprism-lightweight-failure-attribution-for-multi-agent-sys.pdf`
- **证据质量**: medium

## 内容摘要

在基于大语言模型（LLM）的多智能体系统中，失败归因（Failure Attribution）旨在识别导致一次失败执行的具体步骤。
这一任务面临多重固有挑战。

首先，单次执行轨迹可能包含数十个智能体动作和工具调用，步骤之间的因果关系复杂且非线性。
其次，失败的外在表现往往出现在原始错误发生多个步骤之后，这种延迟效应使得简单的尾部回溯难以定位真正的根因。
最后，现有方法往往依赖于昂贵的智能体工作流重放、执行回放，或者在大量合成失败日志上进行有监督训练，这些方法在计算成本、数据获取和实时性方面都存在显著瓶颈。

针对上述挑战，本文提出了 MASPrism，一种利用小型语言模型（SLM）预填充阶段内部信号进行失败归因的轻量级框架。
MASPrism 的核心创新在于完全绕过了传统的解码生成过程，转而挖掘预填充（Prefill）阶段产生的两类关键内部信号：token 级别的负对数似然（Negative Log-Likelihood, NLL）和跨层注意力权重（Attention Weights）。

在预填充阶段，模型需要对输入序列中的所有 token 进行一次性编码。
这一过程中每个 token 的 NLL 反映了模型对该 token 的预测不确定性，而注意力权重则揭示了不同 token 之间的信息依赖关系。
MASPrism 巧妙地利用这两类信号来同时识别「症状步骤」（即表现出异常统计特征的步骤）和「候选故障源」（即可能引发这些症状的更早步骤）。

具体而言，MASPrism 采用两阶段流水线架构。
第一阶段称为 Filtering（症状引导的候选筛选），主要完成四项子任务：
步骤级截断（Step-level Truncation）将过长的执行轨迹压缩到模型上下文窗口可容纳的范围；
步骤级信号提取（Step-level Signal Extraction）计算每个步骤内所有 token 的平均 NLL 和跨步骤的注意力聚合分数；
症状识别（Symptom Identification）基于 NLL 阈值筛选出统计异常的步骤；
候选选择（Candidate Selection）利用从症状步骤反向传播的注意力信号，定位更早的潜在故障源。

第二阶段称为 Diagnosis（多症状故障源排序），包含四个核心步骤：
首先重构一个聚焦的诊断提示（Prompt Reconstruction），将第一阶段筛选出的候选步骤重新组织为诊断上下文；
然后执行第二次预填充以重新计算信号（Signal Recomputation）；
接着对每个候选步骤进行评分（Candidate Scoring）；
最后通过多症状共识机制（Multi-Symptom Consensus）综合多个症状步骤的评分结果，输出最终的故障源排序。

实验部分在 Who&When 和 TRAIL 两个广泛使用的多智能体失败归因基准数据集上展开，涵盖四个评估子集。
MASPrism 使用 Qwen3-0.6B 作为底层小型语言模型，在四个子集中的三个上取得了最佳性能。
在 Who&When-HC 子集上，MASPrism 的 Top-1 准确率相比最佳基线提升了 33.41%。
在 TRAIL 数据集上，MASPrism 甚至超越了包括 Gemini-2.5-Pro 在内的强专有大型语言模型，相对提升最高达 89.50%。

效率方面，MASPrism 处理每条执行轨迹的平均耗时仅为 2.66 秒，相比单轮提示基线实现了 6.69 倍的加速，且整个过程中生成的输出 token 数为零。
这些结果充分证明，MASPrism 为长多智能体执行日志的失败归因提供了一种兼具高准确性和高成本效益的实用框架。

从更广泛的视角来看，MASPrism 的工作具有重要的范式意义。
它表明，轻量级模型在特定诊断任务上的表现可以超越规模远大于自身的专有模型，关键在于找到合适的信号来源和任务分解策略。
预填充阶段的内部信号通常被视为模型运行的「副产品」，而 MASPrism 将其转化为具有明确语义解释的诊断依据。
这一思路不仅适用于失败归因，也可能启发其他类型的智能体系统可观测性研究，例如性能瓶颈定位、异常行为检测和根因分析等方向。

论文还探讨了该方法在实际部署场景中的应用前景，以及如何将排序后的候选故障源集成到调试工作流中以辅助开发者定位问题。
作者特别指出，虽然实验结果令人鼓舞，但在真实生产环境中应用时仍需考虑轨迹长度分布、失败类型占比以及开发者对排序结果的实际使用模式等因素。

## 关键要点

1. **问题定义与现实挑战**
   多智能体系统中的失败归因需要定位导致失败的原始步骤，而非仅识别最终报错位置。
   主要挑战包括：
   - 执行轨迹长且复杂，单次运行可能包含数十个步骤；
   - 故障症状延迟出现，错误证据可能在原始错误后多个步骤才显现；
   - 现有方法成本高昂，依赖重放、回放或合成数据训练，难以扩展。

2. **核心方法：两阶段预填充信号框架**
   MASPrism 采用 Filtering + Diagnosis 的两阶段流水线：
   - **Filtering（症状引导的候选筛选）**：
     对执行轨迹进行步骤级截断；
     提取每个步骤的 NLL 和注意力权重；
     识别症状步骤并筛选候选故障源。
   - **Diagnosis（多症状故障源排序）**：
     重构诊断提示；
     重新计算信号；
     对候选步骤进行评分；
     通过多症状共识机制综合排序。

3. **技术亮点与创新**
   - 利用预填充阶段的内部信号（NLL、Attention）而非解码输出，实现零输出 token 的失败归因；
   - 使用 Qwen3-0.6B 这一轻量 SLM 即可达到超越专有 LLM（如 Gemini-2.5-Pro）的效果；
   - 两阶段设计将长轨迹压缩为少量候选步骤，兼顾效率与准确性；
   - 无需训练或微调，完全基于模型的前向传播信号。

4. **实验结果与性能数据**
   - 在 Who&When-HC 子集上 Top-1 准确率提升 33.41%；
   - 在 TRAIL 数据集上相对 Gemini-2.5-Pro 提升最高 89.50%；
   - 平均处理时间 2.66 秒，相比基线实现 6.69 倍加速；
   - 零输出 token，计算成本极低。

5. **局限与未来方向**
   - 实际部署场景中的内存占用和延迟开销仍需在大规模生产环境中评估；
   - 排序结果如何与开发者调试工作流深度集成有待进一步验证；
   - 方法边界和未来验证方向需要更多真实场景数据支撑，特别是在非英文场景下的表现；
   - 相关工作中涉及 LLM-based 多智能体系统、软件故障定位和轨迹诊断等多个领域，跨领域迁移潜力值得关注；
   - 当前实验集中在特定基准数据集上，开放域智能体系统的泛化能力仍需验证。

## 与综述的关联

MASPrism 与综述中讨论的「智能体可观测性」（Agent Observability）和「故障诊断与归因」（Failure Diagnosis and Attribution）主题高度相关。
当前综述强调，多智能体系统的复杂性使得传统日志分析和简单规则匹配难以有效定位根因，而 MASPrism 提供的轻量级归因方案恰好填补了这一技术和成本之间的空白。

与 AgentTrace、OpenInference 等结构化日志框架相比，MASPrism 不依赖外部追踪基础设施或大量人工标注数据，而是直接从模型内部信号中提取诊断信息。
这种「白盒」诊断思路与综述中提到的可解释性需求形成互补关系。
传统可观测性工具侧重于记录系统外部行为，MASPrism 则展示了如何利用模型内部状态来理解行为背后的原因。

此外，MASPrism 在成本效率上的显著优势（使用仅 0.6B 参数的模型超越千亿级专有模型）也与综述关注的「token 与资源归因」议题深度呼应。
综述指出，多智能体系统的运行成本随步骤数指数增长，而诊断工具本身不应成为新的成本负担。
MASPrism 证明高质量诊断不一定需要昂贵的大型模型推理，这为综述中关于「经济可承受的可观测性」论述提供了强有力的实证支持。

从技术路线角度，MASPrism 的预填充信号提取方法可以与综述中讨论的其他可观测性技术形成组合方案。
例如，将 MASPrism 的诊断输出作为 OpenTelemetry trace 的自定义属性，或者将其候选排序结果集成到 LangSmith、Langfuse 等现有观察平台中，都是值得探索的集成方向。
这种组合思路可以兼顾 MASPrism 的低成本优势和现有平台的可视化、告警能力。

## 我的笔记

MASPrism 最吸引我的地方在于它从根本上重新定义了「诊断」与「推理」之间的关系。
传统观念普遍认为，准确的失败归因需要模型具备强大的链式推理能力，因此研究者和工程师们自然倾向于使用 Claude 3.5 Sonnet、GPT-4、Gemini-1.5-Pro 等大规模模型来完成此类任务。
但 MASPrism 的实验结果令人信服地证明，预填充阶段产生的统计信号——负对数似然和注意力权重——已经蕴含了足够丰富且结构化的诊断信息，足以支撑高质量的失败归因。
这一发现启发我深入思考：在智能体系统的可观测性架构设计中，我们是否过度关注外部输入输出的记录，而忽略了模型内部状态这座潜在的「诊断金矿」？

从工程落地的角度审视，MASPrism 的两阶段流水线设计极具实用价值。
第一阶段通过粗粒度筛选将可能包含数百个步骤的长轨迹压缩为少量高价值候选，第二阶段再对这些候选进行精排，确保最终输出的准确性。
这种「先粗后精」的策略与生产环境中日志检索和异常检测的最佳实践高度一致。
第一阶段的速度优势使得 MASPrism 可以部署在实时或近实时的监控路径上，而第二阶段的精度保证则满足开发者事后调试的需求。
这种分层设计思路对于构建生产级可观测性系统具有直接的借鉴意义。

不过，论文使用 Qwen3-0.6B 作为实验中的 SLM，这一选择在实际部署时仍需审慎评估。
如果业务智能体的执行轨迹以英文为主，或者涉及大量特定领域的术语和工具调用格式，Qwen3-0.6B 的预填充信号是否仍能稳定捕获异常模式，需要额外的验证。
此外，不同模型架构（如纯 Decoder、MOE、Mamba 等）的预填充信号分布可能存在差异，MASPrism 的方法是否具备良好的跨架构泛化能力，也是值得后续研究回答的问题。
对于综述而言，这一局限性恰恰说明「轻量级诊断」领域仍有大量开放问题等待探索。

另一个值得反复品味的细节是多症状共识机制。
在真实生产环境中，一次失败很少由单一根因导致，更多情况下是多个步骤的连锁反应。
当存在多个症状步骤时，每个症状可能指向不同的候选故障源，如何综合这些信号避免遗漏或误报，是失败归因的核心难点之一。
MASPrism 通过注意力权重的跨症状聚合来达成共识，这种设计体现了作者对真实世界故障复杂性的深刻洞察。
对于构建综述中的「故障归因理论框架」而言，多症状共识是一个值得重点引用的技术机制。

对于综述写作而言，MASPrism 是一个不可多得的案例研究。
它雄辩地证明了「轻量级」和「高性能」并非互斥，关键在于找到合适的信号来源和算法设计。
这一论点可以直接支撑综述中关于「可观测性工具应当与业务系统同等级别的成本效率」的论述。
同时，MASPrism 与 AgentRx 等追踪分析工具在方法论上形成了有趣的对照：
AgentRx 强调通过结构化日志和 trace 重建执行全景，MASPrism 则强调通过模型内部信号直接定位异常根源。
两种路线各有适用场景，综述可以将它们并置讨论，为读者呈现更完整的可观测性技术图景。

未来我会持续跟踪该工作是否开源，以及社区是否会在 AutoGen、CrewAI、LangGraph 等主流智能体框架上验证其通用性。
一个特别令人兴奋的想象空间是：如果 MASPrism 的预填充信号提取能够被封装为 OpenTelemetry 的自定义指标或 span 属性，那么它将与现有可观测性生态形成无缝集成，使得「轻量级内部诊断」成为每个智能体系统的标准能力，而不仅仅是研究原型。
综述可以在「未来趋势」一节中专门讨论这种「内部信号 + 外部追踪」的融合方向。
