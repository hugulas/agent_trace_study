# Token Economics for LLM Agents: A Dual-View Study from Computing and Economics

- **标题**: Token Economics for LLM Agents: A Dual-View Study from Computing and Economics
- **作者**: Yuxi Chen, Junming Chen, Chenyu He, Yiwei Li, Yicheng Ji, Yifan Wu, Dingyu Yang, Lansong Diao, Lidan Shou, Hongliang Zhang, Huan Li, Gang Chen
- **年份**: 2026

- **arXiv ID**: 2605.09104
- **类型**: 综述论文（Survey）
- **PDF**: `[s3-011]-token-economics-for-llm-agents-a-dual-view-study-from-comput.pdf`

## 内容摘要

随着大语言模型（LLM）智能体（Agent）技术的快速演进，token 已经不仅仅是模型输入输出的最小语义单元，更演变为 Agentic AI 体系中最核心的经济原语（economic primitives）。

然而，token 消耗量的指数级增长带来了严峻的计算瓶颈、协作摩擦以及安全威胁。

现有的相关调研往往分散在系统优化、架构设计和可信性等各自独立的方向上，缺乏一个统一的框架来评估输出质量与经济成本之间的根本性权衡。

本文首次提出了 Token Economics 这一综合性综述视角，将计算机科学与经济学理论进行深度融合，把 token 同时概念化为生产要素、交换媒介和记账单位三种经济角色，构建了一个跨学科的分析框架。

论文首先从基础概念入手，对 token 进行经济学分类，分析 token 的生产过程与成本结构，并建立了整体 Token Economics 的全景视图。

在此基础上，作者系统性地将经济学理论映射到智能体系统的不同分析层级，形成了从微观到宏观、从功能到安全的完整理论覆盖。

在微观层面（单智能体视角），作者借鉴新古典厂商理论（neoclassical firm theory），将单智能体的 token 消耗建模为预算约束下的要素替代问题。

智能体需要在推理深度、工具调用频率、记忆检索范围等多个维度之间进行权衡，以在有限的 token 预算内最大化任务完成质量。

这涉及到计算与推理效率的优化、记忆架构与上下文管理的精细化设计、工具使用与信息检索的策略选择，以及规划推理过程中的框架治理机制。

具体而言，计算效率可以从 token 密度（token density）、token 数量（token quantity）以及每个 token 的计算和内存开销三个子维度进行优化。

token 密度关注如何在更少的 token 中编码更多信息；token 数量关注如何通过推理压缩和早期终止减少不必要的 token 生成；每个 token 的计算和内存开销则涉及模型架构层面的效率优化，如稀疏注意力、量化推理等技术。

记忆架构需要在长短期记忆的存储成本与检索精度之间取得平衡，避免因过度检索而浪费 token，也避免因检索不足而需要重复推理。

工具调用涉及外部 API 的货币成本与 token 成本的双重考量——某些工具调用不仅消耗时间，其返回结果的大量文本还会显著膨胀后续推理的上下文。

规划与推理的框架治理则关注如何在 Chain-of-Thought、ReAct、Tree-of-Thought 等不同推理范式之间选择最经济的策略，以及如何通过元认知机制在推理过程中动态评估继续推理的边际收益。

在中观层面（多智能体系统视角），作者引入交易成本经济学（transaction cost economics）和委托-代理理论（principal-agent theory），分析多智能体协作中的 token 经济问题。

当多个智能体协同完成复杂任务时，智能体之间的编排调度、通信交互、计算效率以及记忆架构的共享与检索都会直接影响整体的 token 开销。

作者将多智能体协作的经济分析分解为多个子问题：首先是协作消耗的度量与基准测试问题，即如何建立统一的指标来比较不同协作架构的经济效率；其次是智能体编排与调度问题，即如何在多个可用智能体中选择最优的子集并确定执行顺序；第三是智能体通信与交互优化问题，即如何在保证信息传递完整性的前提下最小化通信 token 开销；第四是计算效率问题，在多智能体场景下如何通过并行化和负载均衡降低总体计算成本；第五是记忆架构与检索效率问题，即多个智能体是否应当共享记忆存储以减少冗余检索。

作者特别强调了多智能体系统中的协作摩擦问题：智能体之间的信息交换虽然能够提升任务完成质量，但每一次通信都伴随着 token 消耗，过度的通信会导致成本急剧上升，而通信不足则可能导致任务失败或结果不一致。

委托-代理理论进一步揭示了多智能体系统中的激励对齐难题——当委托方智能体将子任务分配给代理方智能体时，代理方可能出于自身成本最小化的动机而采取次优策略，从而产生委托-代理冲突。

找到最优的通信频率、信息粒度和激励机制，是中观层面 token 经济优化的核心挑战。

在宏观层面（智能体生态系统视角），作者运用机制设计理论（mechanism design）分析智能体生态中的拥堵外部性与定价问题。

当大量智能体同时作为生产者和消费者参与市场时，会出现复杂的经济现象。

作者将生态层面的分析组织为四个维度：生产者-消费者互动涉及智能体服务的定价策略和拥堵管理；生产者-生产者竞争涉及寡头垄断格局的形成与护城河构建；监管者-市场互动涉及外部性的内部化机制；动态 token 生态系统调整则关注市场在供需变化时的自适应演化。

例如，当一个智能体服务平台上的用户数量激增时，模型的响应延迟会增加，这种拥堵效应本质上是一种负外部性，需要通过合理的定价机制（如动态定价、优先队列）或资源分配机制来内部化。

类似地，当少数大型智能体提供商占据市场主导地位时，可能出现寡头垄断定价，损害消费者福利，这需要监管干预或市场机制设计来防范。

在安全维度，作者将对抗性威胁内部化为内生经济约束，提出了贯穿 token 全生命周期的风险分类体系，构建了安全约束下的经济成本模型，并探讨了治理作为经济基础设施的政策含义。

安全风险并非独立于经济考量之外，而是直接增加了 token 消耗的额外成本——输入过滤、对抗性检测、输出审查等安全措施都需要额外的计算资源和 token 预算。

作者将安全风险按照 token 生命周期进行分类：在输入阶段，对抗性输入可能诱导模型产生有害输出；在推理阶段，提示注入和越狱攻击可能绕过安全机制；在输出阶段，模型可能生成偏见、隐私泄露或误导性内容。

每一类风险都需要部署相应的防御机制，而每一种防御机制都带来额外的 token 开销。

最后，论文展望了可微分 token 预算（differentiable token budgets）和动态市场（dynamic markets）等前沿方向，为下一代可扩展智能体系统奠定了理论基础。

可微分 token 预算的核心理念是让 token 消费限制成为模型训练过程中可优化的参数，通过梯度信息自动学习在给定任务上的最优资源分配策略。

动态市场则更进一步，设想了一个 token 可以像货币一样在智能体之间流通和交易的经济环境，为构建大规模、去中心化的智能体经济体提供了理论蓝图。

## 关键要点

1. **Token 的三重经济角色**：本文创新性地将 token 定义为生产要素（production factors）、交换媒介（exchange mediums）和记账单位（units of account），这一概念化超越了传统计算机科学中将 token 仅视为模型输入输出单位的狭义理解，为跨学科研究提供了统一的语义基础。

   作为生产要素，token 是智能体完成认知工作的"原材料"，其数量和质量直接决定了智能体的产出能力；作为交换媒介，token 是智能体之间以及智能体与人类之间信息交换的载体，承载着意图、知识和指令的传递；作为记账单位，token 提供了衡量和比较不同智能体行为经济成本的统一尺度，使得跨系统、跨厂商的成本比较成为可能。

2. **四维分析框架**：
   - **微观层面（单智能体）**：基于新古典厂商理论的预算约束优化，涵盖计算效率、记忆架构、工具调用、规划推理四个子维度。
     智能体被类比为一家在预算约束下寻求产出最大化的厂商，token 则是其需要优化配置的核心生产要素。
     这一层面的分析关注如何在给定预算内实现最优的任务完成质量。
   - **中观层面（多智能体系统）**：基于交易成本与委托-代理理论的协作摩擦最小化，涵盖编排调度、通信优化、计算效率、记忆共享四个子维度。
     多智能体系统被类比为一个企业组织，需要在内部协调成本与外部市场交易成本之间权衡。
     这一层面的分析关注如何通过架构设计降低协作过程中的经济损耗。
   - **宏观层面（智能体生态）**：基于机制设计的拥堵与定价分析，涵盖生产者-消费者互动、生产者竞争、监管互动、动态调整四个子维度。
     智能体生态被类比为一个市场经济体系，需要解决公共品供给、外部性内部化和市场势力规制等经典经济学问题。
     这一层面的分析关注如何设计制度框架以促进生态的可持续繁荣。
   - **安全层面**：将安全风险内部化为经济约束，构建全生命周期的风险分类与经济成本模型。
     安全不再被视为单纯的技术问题，而是经济决策中必须考虑的成本项。
     这一层面的分析关注如何在有限的安全预算内实现最优的风险暴露水平。

3. **核心经济理论映射**：
   - 新古典厂商理论（neoclassical firm theory）→ 单智能体在预算约束下的要素替代与产出优化，关注边际成本与边际收益的平衡。
   - 交易成本经济学（transaction cost economics）→ 多智能体协作中通信与协调的成本分析，关注内部化交易以降低协调成本的机会。
   - 委托-代理理论（principal-agent theory）→ 多智能体系统中激励对齐与行为监督，关注如何设计契约以降低代理成本。
   - 机制设计理论（mechanism design）→ 智能体生态中的定价策略、资源分配与外部性内部化，关注如何设计规则使得自利行为导向社会最优结果。

4. **前沿方向**：可微分 token 预算允许通过梯度下降自动学习最优 token 分配策略，使得智能体能够在训练阶段就内化成本意识。

   具体而言，token 预算可以成为神经网络的一个可学习参数，或者通过强化学习的奖励函数来编码成本约束。

   动态市场则探索 token 作为可交易资源在智能体经济体中的实时定价与流通机制，为构建去中心化的智能体服务市场提供理论基础。

   在动态市场中，token 可能成为一种准货币，智能体可以通过"赚取"和"花费"token 来参与经济交换。

5. **系统性整合价值**：本文的最大贡献在于将此前分散在系统优化、架构设计和安全可信等不同社区的研究工作，首次纳入一个统一的 Token Economics 框架之下，使得研究者可以在同一套经济语言体系中讨论质量-成本的权衡问题。

   这种整合不仅具有理论意义，也为工业界的资源规划和成本管理提供了概念工具。

   在此之前，成本优化、通信效率、安全防御等话题往往被分开讨论，本文证明了它们本质上是同一经济体系的不同表现。

6. **安全与经济成本的统一视角**：论文提出将对抗性威胁纳入经济约束模型，这意味着安全防护的投入不再是一个独立的预算项目，而是应当与功能执行的 token 成本统一核算。

   这一视角对设计成本-aware 的安全策略具有重要指导意义。

   例如，团队可以基于经济分析决定在某些低风险场景下降低安全检测的严格程度，以换取更低的运行成本。

7. **协作摩擦的量化分析**：中观层面的交易成本分析为多智能体系统的效率评估提供了新的维度。

   传统的多智能体评估主要关注任务成功率和完成时间，本文提示我们还应当关注经济效率——一个任务完成得很快但花费了过多 token 的协作模式并非最优。

8. **Token 生产成本的结构性分析**：论文在基础概念部分对 token 的生产过程进行了经济学解构，指出 token 成本不仅包括直接的推理计算成本，还包括数据准备、模型训练分摊、基础设施运维等间接成本。

   这种结构性分析有助于团队更全面地理解 AI 服务的定价逻辑，避免仅关注表面上的每百万 token 单价。

## 与综述的关联

本笔记服务于 agentic trace 可观测性与可靠性综述的写作。

Token Economics 框架为理解 agent 执行轨迹中的成本归因提供了高屋建瓴的理论视角：

- **与成本归因的关联**：在单智能体层面，token 消耗直接对应 trace 中每一步推理、工具调用和记忆检索的经济成本，为 trace-based cost attribution 提供了微观经济学基础。

  综述在讨论如何从执行轨迹中分解和归因成本时，可以引用本文的微观分析框架作为理论支撑。

  具体而言，trace 中的每一次 LLM 调用可以被视为一个生产环节，其输入输出 token 数对应原材料和产出的数量，而模型选择则对应生产技术的选择。

- **与多智能体调试的关联**：在中观层面，多智能体协作中的通信开销和冗余计算可以通过 trace 进行量化观测，进而用交易成本理论解释和优化。

  当综述讨论多智能体系统的故障定位和责任归属时，经济成本可以作为一个客观的度量标准来判断哪些通信是必要的、哪些是冗余的。

  例如，如果两个智能体之间的某次通信对最终输出的贡献微乎其微，但消耗了大量 token，那么这次通信就是一个潜在的优化目标。

- **与基础设施设计的关联**：宏观层面的生态系统分析提示我们，可观测性平台本身也是 token 经济生态的一部分，其定价、拥堵和外部性问题值得在综述中讨论。

  特别是，trace 采集和存储本身也消耗资源，需要在整体经济模型中考量。

  当多个团队共享同一个可观测性平台时，就出现了经典的"公地悲剧"问题——每个团队都有动机最大化自己的数据采集，但 collective 的结果是系统过载。

- **与安全审计的关联**：安全维度提出的经济成本模型表明，安全防护（如对抗性检测、输入过滤、输出审查）本身也消耗 token，需要在 trace 中显式记录安全开销，以实现真正的全成本可视化。

  这与综述中关于安全可观测性的章节直接相关。

  未来的 trace 标准应当定义安全成本的标准化记录格式。

- **与评估基准的关联**：本文提出的 token 消耗度量框架可以作为设计新的 agent 评估基准的维度之一——不仅评估准确率和效率，还评估经济成本，从而推动更具实用价值的基准设计。

  现有的基准如 SWE-bench、HumanEval 等主要关注功能正确性，加入经济维度后将更贴近工业界的实际决策场景。

- **与 Agent 经济学未来的关联**：论文展望的可微分 token 预算和动态市场方向，为综述的前瞻性讨论提供了素材。

  综述可以探讨这些前沿概念对 trace 记录格式、采集策略和分析方法的新要求。

  例如，如果 token 成为可交易资源，trace 系统可能需要记录跨智能体的 token 流转信息。

## 我的笔记

这篇综述的跨学科视角令人耳目一新。

在 agentic trace 分析的研究中，我们通常关注功能正确性和执行效率，但往往忽视了经济维度。

实际上，每一次 LLM 调用、每一次工具使用、每一次记忆检索都有明确的货币成本，而这些成本在执行轨迹中留下了可量化的痕迹。

将经济学理论引入 trace 分析，可以帮助我们回答一些此前难以形式化的问题：为什么某个 agent 选择了更长的推理链而非直接回答？为什么多智能体系统中某些通信是冗余的？安全防护的投入产出比如何衡量？资源分配策略是否实现了帕累托最优？

特别值得关注的是"可微分 token 预算"这一前沿方向。

如果 token 预算可以通过梯度下降进行自动优化，那么 agent 的学习过程将同时考虑任务性能和经济成本，这意味着未来的 agent 架构可能内建成本感知能力。

对于 trace 分析而言，这提出了新的要求：trace 不仅要记录"做了什么"，还要记录"花了多少"以及"为什么这样花"。

可微分预算还可能催生新的研究方向，例如基于强化学习的自适应 token 分配，或者将成本作为正则项纳入 agent 的训练目标函数。

可以想象，未来的 agent 框架可能提供一个"budget-conscious mode"，在该模式下，agent 会在每一步决策前评估继续推理的边际成本与预期边际收益。

此外，本文将安全威胁内部化为经济约束的思路极具启发性。

在当前的 agent 可观测性实践中，安全事件往往被单独记录和分析，但 Token Economics 框架提示我们，安全防护的经济成本应当与功能执行成本统一核算。

例如，一次输入过滤操作消耗的 token、一次对抗性检测增加的延迟成本、一次输出审查导致的额外推理步骤，都应该在 trace 中显式呈现，否则我们无法评估安全策略的真实代价。

这种统一核算的视角也有助于企业在安全投入与业务产出之间做出更理性的权衡。

当安全预算有限时，经济分析可以帮助确定哪些环节的安全投入回报最高。

在中观层面，交易成本理论对多智能体 trace 分析具有直接的指导意义。

当我们观察到两个智能体之间进行了大量高频通信时，交易成本理论促使我们追问：这些通信是否可以通过更好的任务分解或共享记忆来减少？通信的内容是否包含了冗余信息？是否存在"内部化"某些交易以降低协调成本的机会？

这些问题都可以通过分析 agent trace 中的通信模式来回答。

例如，如果 trace 显示两个智能体频繁交换相似的信息，那么引入一个共享记忆层或一个专门的知识中介智能体可能显著降低通信成本。

宏观层面的机制设计视角则启发我们思考可观测性平台自身的经济属性。

当前的主流可观测性厂商（如 Datadog、New Relic）采用基于数据量的定价模式，这实际上创造了一种激励机制——agent 产生的观测数据越多，厂商收入越高。

但这种激励机制是否与用户的成本优化目标一致？

Token Economics 框架提示我们，可观测性市场的制度设计本身也值得研究。

例如，是否可能出现基于"洞察价值"而非"数据量"的定价模式？

在写作综述时，可以将本文作为理论底座，将其他关于成本优化、资源调度和安全审计的实证研究纳入其四维框架中进行组织和评述，从而提升综述的理论深度和系统性。

例如，在讨论单智能体成本优化时，可以引用各类 prompt compression 和 context pruning 的技术工作，并用新古典厂商理论的要素替代框架来解释其经济学逻辑；在讨论多智能体系统的通信优化时，可以引用 agent 编排和消息路由的相关工作，并用交易成本理论来评估其经济价值。

这种理论与实证相结合的方式将显著增强综述的学术深度和实践指导意义。

最后，我认为 Token Economics 作为一个新兴的研究领域，其最大的潜力在于为 AI 系统的工程实践提供一套统一的语言和度量标准。

当前，工程师、产品经理和财务人员在讨论 AI 成本时往往使用不同的术语和指标，导致沟通障碍和决策失误。

Token Economics 框架通过将 token 确立为通用的记账单位，有望消除这种语义鸿沟，使得跨职能团队能够在同一套概念体系下进行成本相关的讨论和决策。
