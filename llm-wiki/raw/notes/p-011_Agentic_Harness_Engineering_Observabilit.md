## 原文摘要翻译

Harness 指模型外部可编辑的工程组件集合，已成为决定代码智能体性能的核心因素，它中介了模型与工具及执行环境之间的交互方式。

然而 harness 工程至今仍是一项手工技艺，因为自动化面临三大障碍：可编辑组件之间的异构动作空间、海量轨迹中可行动信号被淹没、以及编辑效果难以归因。

本文提出智能 Harness 工程（Agentic Harness Engineering，AHE），一个通过三大匹配可观测性支柱解决上述问题的闭环系统。

第一，组件可观测性将每个可编辑 harness 组件赋予文件级表示，使动作空间显式且可回滚。

第二，经验可观测性将数百万原始轨迹标记蒸馏为可逐层下钻的证据语料，使演化智能体能够真正消费这些信息。

第三，决策可观测性将每次编辑与自声明预测配对，并在下一轮任务级结果中加以验证，从而使每次编辑都成为一个可证伪的契约。

实验上，十轮 AHE 迭代将 Terminal-Bench 2 的 pass@1 从 69.7% 提升至 77.0%。

该结果超越人工设计的 Codex harness（71.9%）以及自演化基线 ACE 和 TF-GRPO。

冻结后的 harness 无需重新演化即可迁移。

在 SWE-bench-verified 上取得最高整体成功率，且标记消耗比种子低 12%。

在 Terminal-Bench 2 上跨三个替代模型族带来 +5.1 到 +10.1 个百分点的增益，表明演化出的组件编码的是通用工程经验而非针对特定基准的过拟合调优。

消融实验进一步将增益定位到工具、中间件和长期记忆，而非系统提示。

## 创新点

1. **三大可观测性支柱的统一设计**：首次将组件可观测性、经验可观测性和决策可观测性整合为同一个闭环，分别解决动作空间异构性、轨迹信号淹没和编辑效果归因三大瓶颈。

2. **文件级可编辑 harness 基底**：基于 NexAU 框架将 harness 解耦为七种正交组件类型，包括系统提示、工具描述、工具实现、中间件、技能、子智能体配置与长期记忆，每个失败模式均可直接映射到单一组件类，实现文件级差异对比与回滚。

3. **可证伪的变更契约**：通过变更清单要求演化智能体为每次编辑自声明预测（预期修复的任务与可能回退的任务），下一轮通过任务级成败差异进行验证，不成立的编辑在文件粒度上回滚。

4. **跨基准与跨模型族迁移**：演化后的冻结 harness 在不重新训练的情况下迁移到 SWE-bench-verified 和三个不同模型族，证明 harness 级别的演化捕获的是通用协调模式。

5. **对自演化极限的清醒分析**：发现组件之间存在非加性交互，且循环对修复的归因可靠但对回退的预见几乎盲目，为后续研究指明了方向。

## 一句话总结

AHE 通过三大可观测性支柱，在固定基座模型的条件下将代码智能体的 harness 转化为可自主演化的外表面。

十轮迭代将 Terminal-Bench 2 的 pass@1 从 69.7% 提升至 77.0%，且演化后的 harness 可跨基准与跨模型族迁移。

## 研究问题

代码智能体在长程软件工程任务上的进步不仅依赖于底层语言模型，同样依赖于周边工程组件。

这些周边工程组件包括塑造工作风格的系统提示、暴露文件系统与命令行接口的工具、以及控制上下文与执行的中间件。

在英文文献中，这些组件统称为 harness。

Harness 设计即使在基座模型固定的情况下也能显著改变长程代码基准的任务完成率，使 harness 工程成为提升代码智能体的一等杠杆。

然而，最优 harness 具有模型特异性：为一个基座模型调优的 harness 往往在另一个模型上表现不佳，必须在基座模型更换时重新适配。

当前实践中，这种适配完全依赖手工。

开发者检查轨迹、识别重复失败模式、再手动在提示、工具、中间件和技能之间进行编辑。

随着基座模型快速迭代，这种手工循环越来越难以跟上，造成模型能力与所需 harness 之间的鸿沟不断拉大。

一个自然方向是用演化智能体基于经验自动优化 harness 组件，但现有方法很少联合演化全部可编辑组件，大多数只聚焦单一组件，如提示、技能或上下文剧本。

联合演化多组件端到端面临两个结构性障碍。

一是长而无结构的轨迹几乎不产生可行动的信号。

二是紧耦合的 harness 框架使得提示之外的编辑极易出错。

因此本文提出核心问题：如何使演化智能体联合且稳定地演化代码智能体 harness 的全部可编辑组件？

## 数据与任务定义

**优化目标基准**：Terminal-Bench 2，共 89 个任务，按官方难度分为 Easy（4 个）、Medium（55 个）、Hard（30 个），每任务超时时间延长至 1 小时。

**跨基准迁移评估**：SWE-bench-verified，共 500 个任务，覆盖七个代码仓库。

**评估指标**：
- **pass@1**：每任务 $k$ 次运行的平均二元成功率；
- **tokens/trial**：每次试验中所有大语言模型调用的提示加补全标记均值，以千为单位。

基础设施中断或超时的试验在 pass@1 中计为失败（与官方排行榜一致），但在标记均值中排除以避免截断数据。

**基座模型**：演化循环与主实验中的三个角色智能体（编码智能体、调试智能体、演化智能体）共享同一基座模型 GPT-5.4 high。

跨模型迁移实验额外评估五种替代基座。

其中包括 GPT-5.4 medium 与 GPT-5.4 xhigh。

另有 qwen-3.6-plus、gemini-3.1-flash-lite-preview 与 deepseek-v4-flash。

## 方法主线

AHE 将 harness 优化转化为由另一个智能体驱动的闭环，基座模型保持固定，仅显式编辑 harness。

设计原则是循环的每个阶段都必须可观测。

AHE 忠实记录每个阶段产出的产物，并以结构化、分层的方式表示，使另一个智能体能够读取并行动。

三大可观测性层实现这一原则。

### 组件可观测性：解耦的文件级 harness 基底

Harness $H$ 实例化在 NexAU 框架上，该框架将七种正交组件类型暴露为单一工作空间中的显式文件。

这七种类型包括系统提示、工具描述、工具实现、中间件、技能、子智能体配置与长期记忆。

各组件类型之间松耦合，添加中间件无需编辑系统提示，添加技能无需触碰任何工具。

这种解耦实现了组件可观测性。

每个失败模式均可直接映射到单一组件类，为演化智能体提供清晰的动作空间，并将每次通过率变化定位到单个文件，而非散落在数百行无结构的提示散文中。

每次逻辑编辑自动成为工作空间版本历史中的一次提交，天然获得文件级差异与回滚粒度。

种子 harness $H_0$ 被刻意设计为极简：仅包含单个命令行执行工具，无中间件、无技能、无子智能体。

这样可确保后续每个组件的增益都能被准确归因。

### 经验可观测性：分层轨迹证据语料

对基准中每个任务，使用当前 harness 生成 $k$ 条轨迹。

原始轨迹可能包含数百万标记的无结构消息。

为实现经验可观测性，AHE 应用 Agent Debugger 框架，将轨迹组织为可导航的文件化环境。

每条轨迹消息驻留于独立文件中，通过通用命令行与脚本工具访问。

同一查询的轨迹被置于同一环境，调试器需分析失败根因或成功模式，生成逐任务分析报告。

所有报告进一步聚合为单个基准级概览文档，作为每轮迭代的入口点。

此外，原始轨迹也以原始形式和轻度清洗形式提供，支持智能体在需要时验证报告中的声明。

所有内容均以文件形式提供，实现渐进式披露，既节省标记又提升决策质量。

### 决策可观测性：可审计的证据驱动编辑

演化智能体闭合 AHE 循环。

每轮迭代中，它读取分层证据语料，决定新增、修改或删除哪些 harness 组件，将编辑应用于工作空间，并记录每次编辑的推理依据。

两项约束共同实现决策可观测性。

**可控性**：演化智能体只能在工作空间内写入，运行目录、追踪器、验证器和大语言模型配置均为只读，且种子系统提示标记为不可删除。

这些限制阻断了无约束自修改器可能采取的捷径，如禁用验证器、替换模型或提高推理预算，确保所有记录到的增益都可归因于 harness 编辑。

**证据驱动变更与记录预测**：每次编辑附带一个清单条目，列明失败证据、推断根因、针对性修复措施以及预测影响。

该清单是循环的证据账本。

下一轮中，循环将预测修复集与预测回退集和观测到的任务级差异求交，产生逐编辑裁决。

每次编辑由此成为下一轮可证伪的契约，取代了 rationale-driven 的自我辩护。

### 机制流程

AHE 的每一轮迭代遵循以下流程：

1. **轨迹生成与压缩**：将当前 harness $H_{t-1}$ 与基座模型 $M$ 送入基准，为每个任务生成 $k$ 条原始轨迹，再清洗掉冗余内容并压缩为规范格式，得到中间轨迹 $\widetilde{T}_t$。

2. **归因对齐与回滚**：将上一轮变更清单的预测与本轮观测到的任务级成败差异进行对齐，验证每个编辑的预测是否成立，对未通过的编辑执行文件级回滚，得到更新后的 harness。

3. **证据提取与蒸馏**：将清洗后的轨迹输入调试智能体，逐任务提取失败根因与成功模式，再融合为基准级概览，输出结构化证据语料 $R_t$。

4. **组件更新与提交**：演化智能体读取证据语料与归因结果，对 harness 组件进行新增、修改或删除操作，记录变更清单 $C_t$，并以版本标签形式提交本轮迭代。

每轮迭代在 pass@1 严格超过历史最优时，将当前 harness 提升为 $H_{\text{best}}$。

### RQ1：主结果

在 Terminal-Bench 2 上运行十轮 AHE 迭代（从仅含命令行的 NexAU$_0$ 种子出发，耗时约 32 小时），最佳配置记为 AHE。

两个自演化基线 ACE 和 TF-GRPO 也从同一 NexAU$_0$ 种子出发。

|--------|----------|----------|-----------|-----------|
| **Human-designed harness** | | | | |
| OpenCode | 47.2% | 75.0% | 52.7% | 33.3% |

AHE 超越所有基线。

按难度看，唯一例外是 Hard 层级，AHE 略低于 Codex。

该差距源于 AHE 组件在长程任务上的相互干扰，而非能力缺失。

ACE 蒸馏自然语言剧本供智能体上下文读取，TF-GRPO 是强化成功工具序列的轨迹反馈变体。

两者均不编辑周边脚手架。

AHE 则联合演化系统提示、工具、中间件和长期记忆，增益集中于后三者。

这正是 ACE 和 TF-GRPO 未触及的层。

### RQ2：跨基准与跨模型迁移

AHE 的 harness 在 Terminal-Bench 2 上以 GPT-5.4 high 演化。

为检验其是否编码通用经验，将演化后的 harness 冻结后不经重新演化直接迁移到两种离靶场景。

**跨基准迁移**：ACE 和 TF-GRPO 在整体成功率上均低于 NexAU$_0$ 种子，同时标记消耗比种子多 11% 到 29%。

AHE 则取得最高整体成功率，种子相对增益集中在 django 和 sphinx-doc。

这两个仓库最大且标记消耗最高，其多步编辑验证循环的结构与 AHE 在 Terminal-Bench 2 上压缩的工具、中间件和长期记忆相匹配。

AHE 还将整体标记降低 32%（相对 ACE）、21%（相对 TF-GRPO）和 12%（相对种子）。

将行为编码到工具、中间件和记忆中，避免了仅调提示的基线每轮重新推导的成本。

**跨模型迁移**：在五种替代基座上重新评估 NexAU$_0$ 种子和 AHE。

跨模型族增益显著高于同族内增益。

deepseek-v4-flash 提升 +10.1 pp。

qwen-3.6-plus 提升 +6.3 pp。

gemini-3.1-flash-lite-preview 提升 +5.1 pp。

以上增益均高于 GPT-5.4 medium 与 xhigh 的 +2.3 pp。

这表明离饱和越远的基座越依赖 AHE 固定在工具、中间件和长期记忆中的协调模式，而更强的基座能以低边际成本从提示中重新推导相同协调。

**自归因可靠性**：

每轮演化中，演化模型的变更清单会声明下一轮预期修复哪些任务以及哪些任务可能回退。

将第 $N-1$ 轮预测与第 $N$ 轮真实结果对比，分别计算修复与回退的精确率和召回率。

修复预测的平均精确率为 33.7%，召回率为 51.4%，约为随机基线的五倍。

每次 harness 编辑都落在智能体能够预见的真实目标上。

回退预测则几乎盲目。

平均精确率仅 11.8%，召回率 11.1%，约为随机基线的两倍。

智能体能论证编辑为何有帮助，但无法可靠命名同一条编辑即将破坏的任务。

这正是演化曲线中出现非单调步进的根源。

缩小这一差距是后续自演化循环最清晰的方向。

## 深度分析

**可观测性作为设计支点**：

本文的核心洞察是，harness 演化的瓶颈不在于智能体能力，而在于可观测性。

一旦演化智能体获得清晰的动作空间与结构化上下文，它就能可靠收敛到更优的 harness 设计。

三大可观测性支柱分别对应了自动化 harness 工程中的三个经典难题。

组件可观测性通过文件级解耦解决了异构动作空间问题。

经验可观测性通过分层蒸馏解决了轨迹信号淹没问题。

决策可观测性通过自声明预测与验证解决了编辑归因问题。

这种设计将 harness 演化从试错转变为基于可证伪契约的系统性工程。

**Harness 演化对比模型训练**：

本文将 harness 级别演化定位为模型侧训练的互补轴。

模型训练改进的是基座能力，而 harness 演化改进的是模型与环境的协调方式。

一个外部化、可审计的表面使代码智能体的经验得以积累，且不会随模型更换而失效。

前提是 harness 编码的是结构性协调模式而非散文级策略。

**组件交互的非加性**：

消融实验揭示了一个重要现象。

单组件增益之和（+11.1 pp）超过完整 AHE（+7.3 pp）。

这是因为记忆、中间件和系统提示都趋向同一类闭合式验证，堆叠后会在长程预算内产生冗余重验证。

演化智能体以 Medium 任务为主导进行优化，因此收敛到一个 Medium 偏重的权衡，牺牲了部分 Hard 上的记忆效果。

这一发现提示未来工作需引入交互感知的演化策略。

**回退盲视的启示**：

循环对修复的预见相对可靠（约 5 倍随机基线），但对回退的预见几乎与随机无异（约 2 倍随机基线）。

这意味着智能体擅长解释为何这个编辑应该有用，但不擅长预测这个编辑会破坏什么。

这种不对称性是所有自演化系统的共同风险。

乐观偏见导致过度编辑，随后产生不可预见的回退。

**标记效率的意外收获**：

AHE 不仅在成功率上超越基线，还显著降低了标记消耗。

将行为编码到工具、中间件和记忆中，避免了仅调提示的基线在每次调用时重新推导策略的成本。

这一发现对实际部署尤为重要。

更好的 harness 不仅更准确，也更经济。

## 局限

**基准范围**：

评估仅在 Terminal-Bench 2 上驱动演化，并在 SWE-bench-verified 上探测迁移。

尽管冻结 harness 能迁移到第二个任务表面和三个替代模型族，但更广泛的编程语言、仓库规模部署和人机协同工作流尚未测试。

**演化操作点耦合**：

AHE 的步数预算和每任务超时在演化期间针对 GPT-5.4 high 拟合，因此跨模型迁移数字混杂了 harness 可迁移性与操作点耦合。

同一家族内增益在推理层级之间呈非单调。

解耦这些因素需要在多个操作点下重新运行循环。

**自修改治理**：

AHE 将编辑限制在工作空间内，在版本化清单中归因每次变更，并以文件粒度回滚无效编辑，但尚未提供完整的防护栏栈。

长程 harness 清理和更强的滥用预防仍不完整，AHE 应被视为受控研究原型，而非完全成熟的自主自改进系统。

## 我的笔记

AHE 为代码智能体的 harness 工程提供了一个极具实践价值的自动化路径。

其核心贡献不仅是性能数字，更是可观测性优先的设计哲学。

与其假设智能体无所不能，不如为智能体提供结构化、可消费、可验证的环境。

三大支柱中，决策可观测性尤其具有方法论意义。

它将传统提示工程中的试错调参转化为假设验证回滚的科学流程。

从复现角度，AHE 对基础设施的要求较高。

需要支持沙盒执行、版本化、多轮迭代调度的框架，以及能够处理百万级标记轨迹的清洗与蒸馏流水线。

NexAU 框架的文件级解耦设计是关键使能器，若要在其他智能体框架上复现，首要任务是确保 harness 组件具备类似的正交性与文件级可编辑性。

从研究延伸角度，最值得关注的问题是回退盲视。

如果未来能结合静态分析或形式化验证来预测编辑的副作用，将有望显著提升自演化循环的稳定性。

此外，组件交互的非加性也提示 harness 设计可能需要从贪心叠加转向联合优化。

例如引入多目标演化或交互感知奖励塑形。
