# Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey

## 核心信息

- 标题: Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey
- 作者: Yasmin Moslem, John D. Kelleher
- 年份: 2026
- 会议/期刊: Zenodo (CERN European Organization for Nuclear Research)
- DOI: 10.5281/zenodo.18701861
- 论文链接: http://arxiv.org/abs/2603.04445v2

## 原文摘要翻译

大型语言模型（LLM）在能力、成本和适用领域方面呈现快速多元化增长，这使得推理阶段的智能模型选择成为一项关键需求。小型模型足以处理常规查询，而复杂任务则需要能力更强的模型。然而，静态模型部署并未考虑输入查询的复杂度和领域差异，导致性能次优且成本上升。基于查询特征自适应选择模型的动态路由系统已成为应对这一挑战的解决方案。本文对最先进的多 LLM 路由与级联方法进行了系统分析。与在单一模型内部进行路由的混合专家（Mixture-of-Experts）架构不同，本文研究的是跨多个独立训练 LLM 的路由。我们涵盖了多种路由范式，包括基于查询难度、人类偏好、聚类、不确定性量化、强化学习、多模态以及级联的方法。针对每种范式，我们分析了代表性方法并考察了关键权衡。除了分类学视角之外，本文还提出了一个概念框架，从三个维度刻画路由系统：决策何时做出、使用什么信息、以及如何计算决策。这一视角强调，实际系统往往是组合式的，在运行约束下整合多种范式。我们的分析表明，有效的多 LLM 路由需要在相互竞争的目标之间取得平衡。最优路由策略的选择取决于部署环境与计算约束。设计良好的路由系统能够通过策略性地利用各模型的专业化能力，在最大化效率增益的同时，超越甚至是最强大的单一模型。与此同时，开发能够跨不同架构、模态和应用泛化的路由机制仍然是开放的挑战。

## 创新点

1. **系统化的多 LLM 路由分类学**：本文首次将跨独立训练 LLM 的动态路由与级联方法整合为统一的分类体系，明确区别于单模型内部的混合专家架构。现有工作分散于不同社区，本文将它们归纳为六大范式（难度感知、偏好对齐、聚类、强化学习、不确定性、级联），为后续研究提供了清晰的坐标系。

2. **三维概念框架**：提出从“决策时机”“使用信号”“计算方式”三个正交维度刻画任意路由系统的设计空间。这一框架揭示了实际系统往往是组合式的。例如 AutoMix 同时使用了多阶段结构、响应级信号和启发式与监督学习的混合计算方式。

3. **对路由与级联的互补性分析**：不仅分别讨论了路由（单次决策）和级联（逐级提升），还分析了生产系统中将两者组合使用的趋势。论文指出，路由适合需要低延迟的场景，而级联适合对质量有严格保证的场景，两者的组合可以进一步优化成本与性能之间的权衡边界。

4. **大规模基准与量化评估视角的整合**：系统梳理了多个专用评测基准，并明确区分了任务性能指标与系统效率指标，强调了帕累托前沿作为联合评估工具的不可替代性。

## 一句话总结

本文系统综述了跨独立 LLM 的动态路由与级联方法，提出三维概念框架，指出组合式路由系统能够在效率与质量之间取得优于任何单一模型的帕累托表现，同时揭示了跨架构泛化与多目标统一优化等开放挑战。

## 研究问题

生产环境中部署 LLM 时存在一个根本性的成本与性能两难：
- 若所有查询都交由最强大的模型处理，简单查询会浪费不必要的计算资源和成本；
- 若使用较小的模型统一处理，复杂查询可能超出其能力边界，导致输出质量下降。

查询的复杂度分布极不均匀，从简单的事实性问题到需要多步推理的复杂任务差异巨大。静态部署策略无法根据输入查询的动态特征进行自适应调整。因此，核心研究问题是：如何在推理阶段根据查询特征自适应地选择或组合多个具有不同能力、成本和专业化的模型，以在保持或提升输出质量的同时最小化推理成本？

此外，论文还关注以下子问题：
- 如何有效估计查询难度或模型对特定查询的胜任程度？
- 如何利用人类偏好数据训练具有跨领域泛化能力的路由器？
- 在多模型级联场景中，如何设计可靠的质量估计机制以决定何时向上级模型转移？
- 如何在质量、成本、延迟等多个相互竞争的目标之间进行系统性的权衡与优化？

## 数据与任务定义

### 评测基准

论文系统梳理了多个专门用于评测 LLM 路由系统的基准数据集：

- **RouterBench**（Hu et al., 2024）：包含超过 40.5 万条预计算推理输出，覆盖 11 个能力各异的 LLM 和 7 个任务。数据集附带详细的性能和成本元数据，支持在真实约束下对路由策略进行系统分析。

- **RouterEval**（Huang 等人，2025）：大规模开源路由评测基准，包含超过 2 亿条性能记录，覆盖 8500 多个大语言模型和 12 个广泛使用的基准测试。该基准将路由建模为 $m$ 路分类问题，支持 $m$ 取 3、5、10、100、1000 等不同难度级别，并支持全强模型组、全弱模型组和混合模型组的评估。

- **MixInstruct**（Jiang et al., 2023）：面向指令遵循任务的评测基准，聚合了 11 万个来自多个指令数据集的样例，通过 LLM 对比获取成对偏好标签，用于评估基于偏好监督的路由和集成方法。

- **LLMRouterBench**（Li et al., 2026）：包含超过 40 万个实例，覆盖 21 个数据集和 33 个模型。该基准同时提供面向性能的路由指标和面向性能与成本权衡的路由指标，并集成了 10 个路由基线方法。

### 任务设置

路由任务可形式化为：给定查询 $q$ 和候选模型池 $\mathcal{M} = \{M_1, M_2, \ldots, M_n\}$，选择一个（路由）或序列（级联）模型，使得在性能约束下成本最小化，或在成本约束下性能最大化。

### 评估指标

- **任务性能指标**：根据领域不同使用特定指标，如多项选择题的准确率、事实查询的精确匹配、代码生成的 pass@k、机器翻译的 chrF 或 COMET 分数。
- **成本指标**：通常相对于强基线报告（例如成本仅为 GPT-4 的 24%，质量却达到其 97%），或直接以货币价值计量。
- **效率指标**：延迟（端到端响应时间，以秒为单位）和吞吐量（每秒处理的 token 数 TPS 或每秒查询数 QPS）。延迟与吞吐量之间存在固有的权衡关系。
- **联合评估**：通过帕累托前沿可视化质量与成本的可行组合，有效的路由系统应支配任何单一模型。

## 方法主线

本文将方法组织为六大路由范式，并通过三维概念框架进行统一刻画。

### 三维概念框架

论文提出的概念框架从以下三个维度分析路由系统：

1. **决策时机**：
   - **生成前决策**：在生成任何输出之前选择模型，完全依赖查询本身的特征。
   - **生成后决策**：在小模型生成响应后，根据响应质量决定是否接受或升级。
   - **多阶段决策**：采用顺序升级结构，可包含多个生成前和生成后决策。
   - **在线自适应**：根据部署过程中的实时反馈动态调整策略。

2. **使用信号**：
   - **查询级信号**：仅使用输入查询文本。
   - **模型元数据**：使用模型的能力标签、评分、领域专长等信息。
   - **响应级信号**：使用模型生成的响应内容进行质量估计。
   - **反馈信号**：利用历史交互数据或下游任务性能进行累积学习。

3. **计算方式**：
   - **启发式**：基于固定规则或阈值。
   - **监督学习**：训练分类器或回归模型预测最佳模型。
   - **赌博机算法**：通过在线探索与利用的平衡学习路由策略。
   - **强化学习策略**：使用 PPO、GRPO 等算法优化长期回报。

### 六大路由范式

1. **难度感知路由**：基于估计的查询复杂度分配模型。代表方法 BEST-Route 使用基于 DeBERTa-v3-small 的多头路由器估计查询难度，并选择最优模型和采样策略。若小模型输出不达标，则增加采样次数或将查询路由到更大模型。Semantic Router 使用 ModernBERT 分类器分析查询意图和复杂度，将需要推理的查询路由到支持思维链的模型。

2. **偏好对齐路由**：利用人类反馈的偏好数据训练路由器。RouteLLM 在人类反馈竞技场与裁判模型的增强数据集上训练矩阵分解路由器和因果大语言模型路由器。其中矩阵分解路由器在多轮对话基准上表现最优且最具成本效益。Arch-Router 将路由策略作为输入上下文提供给 15 亿参数模型，使其根据对话历史和用户偏好选择模型。

3. **聚类路由**：通过无监督学习对相似查询分组。UniRoute 在训练集上应用 K-means 聚类找到 K 个质心，将验证集划分为代表性簇。每个测试时 LLM 表示为 K 维特征向量（各簇误差），然后路由到在该查询所属簇上成本调整后误差最小的模型。

4. **强化学习路由**：通过策略优化或赌博机方法学习路由决策。Router-R1 和 R2-Reasoner 使用 PPO 在问题与得分对数据集上训练路由器。Dueling Feedback 将问题建模为上下文决斗赌博机，从成对偏好比较中学习。TI-UCB 使用上置信界策略平衡探索与利用。

5. **不确定性路由**：基于模型置信度估计进行路由。CP-Router 使用共形预测构建校准的置信区间。Probe-based UQ 通过探针模型估计基础模型的不确定性。Self-REF 使用自验证机制评估输出质量。

6. **级联**：顺序尝试从小到大的模型，仅在质量估计不达标时升级。FrugalGPT 是最具代表性的级联方法。Cascade Routing 采用迭代方式在每一步选择最佳模型，允许跳过或重新排序模型。AutoMix 结合路由与级联，包含 LLM 路由器、基于阈值的质量估计器和停止判断器三个组件，实现三步流程：小模型生成、自验证、基于置信度的路由升级。LM-Blender 通过交叉注意力对候选输出进行排名和融合。

### 机制流程

以典型的生产级组合路由系统为例，核心推理流程可概括为以下三步：

1. **输入查询分析与特征提取**：
   - 输入：用户查询 $q$ 和候选模型池 $\mathcal{M}$ 的元数据。
   - 操作：路由器提取查询的语义嵌入（如通过 BERT 系列编码器），同时检索历史数据中相似查询的性能记录。若系统支持多模态，则同时处理文本、图像等输入信号。
   - 输出去向：查询表示向量和相关历史信息传递至路由决策模块。

2. **路由决策与模型分配**：
   - 输入：查询表示、各模型在历史数据上的性能与成本估计、用户定义的质量阈值或预算约束。
   - 操作：决策模块根据系统架构选择计算方式。若是监督式路由器，则运行分类器预测最佳模型。若是级联系统，则按预设顺序启动最小模型。若是在线自适应系统，则结合赌博机策略和实时反馈调整选择概率。同时评估查询难度或模型不确定性，以决定是否采用 best-of-n 采样或多阶段升级。
   - 输出去向：选定的目标模型 $M^*$ 或模型执行序列 $\langle M_1, M_2, \ldots \rangle$，以及采样参数（如温度、top-p）。

3. **响应生成、质量验证与反馈回传**：
   - 输入：目标模型生成的候选响应 $r$。
   - 操作：对于单阶段路由系统，直接将响应返回给用户。对于级联或多阶段系统，运行质量估计器（如基于奖励模型的评分、自验证提示或共形预测区间）判断响应是否满足阈值。若不满足，则将查询连同中间结果升级至下一级模型。最终将选定的响应输出，并将查询、模型、性能记录写入反馈缓冲区用于在线学习。
   - 输出：返回给用户的高质量响应，以及用于更新路由器参数的交互日志。

## 关键结果

由于本文是综述性质，核心结果体现在对现有方法性能特征的系统性归纳和基准规模的量化描述上：

- **基准规模**：RouterBench 提供 40.5 万条预计算输出覆盖 11 个 LLM 和 7 个任务。RouterEval 达到 2 亿条性能记录、8500 多个 LLM、支持最高 1000 路分类。LLMRouterBench 覆盖 40 万实例、21 个数据集、33 个模型并集成 10 个基线。

- **路由器性能**：在多轮对话基准上，矩阵分解路由器和因果大语言模型路由器在人类反馈竞技场与裁判模型的增强数据集上训练后表现极具竞争力。其中矩阵分解路由器在效率和成本效益方面最优。大规模路由评测基准的关键发现表明，有能力的路由器能够通过模型互补性实现超越最佳单一模型的性能。

- **成本节省**：SCOPE 框架在保持竞争力的同时实现了 84.46% 的 API 成本节省。典型的成本与质量权衡报告形式如：成本仅为 GPT-4 的 24%，质量却达到其 97%。

- **帕累托优势**：有效的路由系统应在质量与成本帕累托前沿上支配任何单一模型。论文强调，设计良好的路由系统不仅能节省成本，还能通过策略性利用不同模型的专业化能力，在整体性能上超越最强大的单一模型。

## 深度分析

### 框架的洞察价值

三维框架的最大价值在于揭示了路由系统设计的组合本质。Table 1 显示，几乎所有实用系统都不是单一范式的纯粹实现。例如 BEST-Route 结合了监督学习分类器和启发式阈值规则，同时使用了查询级信号和响应级验证。AutoMix 则是多阶段结构、响应级自验证以及启发式与监督混合决策的典型组合。这一框架帮助研究者和工程师将现有方法解构为可重组的模块，从而根据部署约束灵活组合。

### 范式间的权衡

六大范式在信号获取成本、决策延迟和泛化能力上存在显著差异：
- **难度感知路由**的决策延迟最低（纯前向分类），但对查询难度的估计本身可能出错，且难以捕捉模型在特定细分领域的专长差异。
- **偏好对齐路由**能够利用人类反馈的丰富信号，但偏好数据的获取成本高，且跨领域泛化能力受限于训练数据的多样性。
- **聚类路由**无需标注数据，依赖无监督表示，但簇的质量直接决定路由效果，且对新查询的分配可能不稳定。
- **强化学习路由**理论上能优化长期回报，但训练不稳定，样本效率低，在实际部署中需要谨慎处理探索与利用的权衡。
- **不确定性路由**提供了理论上可校准的置信估计，但不确定性量化方法在不同模型家族间迁移困难。
- **级联**能够在保证质量的同时最大化小模型的利用率，但顺序生成导致延迟累积，且质量估计器的误差会直接决定整体效率。

### 评测基准的演进与挑战

从 RouterBench 到 RouterEval 再到 LLMRouterBench，评测基准在规模和覆盖度上迅速扩张：模型数量从 11 个增加到 8500 个以上，分类粒度从固定任务扩展到 $m$ 路动态分类。然而，基准的膨胀也带来了新的问题：RouterEval 的 2 亿条记录虽然提供了统计上的充分性，但是否能反映真实部署中的查询分布仍存疑。此外，现有基准多聚焦于文本模态，对多模态路由的评测支持不足。

### 关键数字的解读

SCOPE 报告的 84.46% API 成本节省是一个引人注目的数字，但需要注意到这是在特定任务和模型池设定下的结果。论文中提到的成本仅为 GPT-4 的 24% 而质量达到其 97% 这类相对指标虽然直观，但高度依赖于基线选择。若以更强或更弱的模型为参照，同一系统的相对表现会有很大差异。因此，帕累托前沿作为联合评估工具比单一数字报告更具信息量。

## 局限

1. **综述性质导致的方法深度限制**：作为综述论文，本文对每个具体方法的描述受限于篇幅，无法深入展开训练细节、超参数设置和完整的消融实验。例如对 BEST-Route 的 best-of-n 采样策略，论文仅做了概念性介绍，未报告采样次数 $n$ 的具体取值范围和不同取值对成本与质量曲线的影响。

2. **缺乏统一的实验对比**：由于涵盖的方法数量庞大且实验设置各异，本文未能提供一个在相同模型池、相同查询集上的大规模对照实验。读者难以直接比较不同范式的绝对性能差异，只能从各自原论文的报告中推断相对优劣。

3. **三维框架的粒度限制**：三维框架虽然具有概念上的清晰性，但在某些边界情况下的区分度不足。例如在线自适应既可以描述决策时机的动态性，也可以描述计算方式的在线学习属性，两者之间的界限在复杂系统中可能模糊。

4. **对实际系统约束的讨论不足**：论文虽然提到了延迟与吞吐量的权衡，但对工程部署中的其他关键约束——如模型加载的内存开销、不同模型间的并发调度、KV Cache 的共享与隔离——讨论较少。这些因素在生产环境中往往对路由策略的可行性具有决定性影响。

5. **开放挑战的共识性大于建设性**：论文列出的开放挑战（如跨架构泛化、多模态路由、统一多目标优化）大多是领域共识，缺乏对这些挑战为何难以解决的技术根源分析，以及可能的突破路径建议。

## 我的笔记

### 与相关工作的联系

本文与 MoE 架构的路由研究形成有趣的对照。MoE 的路由发生在单模型内部、训练阶段协同优化，而本文关注的是跨独立模型、纯推理阶段的动态选择。两者可以互补：MoE 解决的是单模型参数效率问题，本文解决的是多模型部署的经济效率问题。一个自然的问题是：是否可以在 MoE 框架中引入跨模型路由，即一个超级路由器在不同 MoE 模型之间进行选择？这可能会成为未来的一个研究方向。

### 工程实践启示

对于实际部署多模型服务的团队，本文的三维框架是一个极好的需求分析工具。在选择路由方案前，可以先明确：
- **决策时机**：业务能否容忍级联带来的延迟累积？若不能，则优先生成前路由。
- **使用信号**：是否有历史交互数据可用？若有，偏好对齐或强化学习路由值得投入；若数据稀缺，聚类或不确定性路由更为现实。
- **计算方式**：团队是否有足够的机器学习工程能力维护在线赌博机或强化学习系统？若资源有限，监督式分类器加启发式规则的混合方案是最稳健的起步选择。

### 值得关注的具体方法

- **矩阵分解路由器**（RouteLLM）：在多个基准上表现优异且计算效率最高，适合作为强基线。
- **UniRoute**：无监督聚类思路在冷启动场景下特别有吸引力，无需标注即可建立初始路由策略。
- **AutoMix**：三步流程（生成、验证、升级）清晰可落地，其 POMDP 建模方式也为级联系统的理论分析提供了形式化基础。

### 未来研究方向

- **统一多目标优化**：当前系统多通过固定权重或约束参数平衡质量、成本、延迟，真正的多目标优化（如将三者同时作为可调目标）仍待探索。
- **跨模态路由**：随着多模态 LLM 的兴起，如何在文本、图像、音频混合查询场景下进行有效路由是一个新兴问题。
- **在线学习与分布偏移**：实际部署中查询分布会随时间变化（概念漂移），现有路由系统的鲁棒性评估和自适应更新机制研究不足。
