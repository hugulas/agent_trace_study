## 原文摘要翻译

基于大语言模型的多智能体系统已成为使自主智能体解决复杂任务的重要范式。随着这些系统在复杂度上不断扩展，成本成为实际部署中必须考虑的重要因素。然而，现有工作很少探讨如何在显式预算约束下构建多智能体系统。本文提出 BAMAS，一种构建具有预算意识的多智能体系统的新方法。BAMAS 首先通过将大语言模型选择问题建模为整数线性规划问题来选取最优模型集合，从而在性能和成本之间取得平衡。随后，它利用基于强化学习的方法确定这些模型应如何协作，以选择交互拓扑结构。最后，系统基于所选智能体及其协作拓扑进行实例化并执行。我们在三个代表性任务上评估了 BAMAS，并与最先进的智能体构建方法进行比较。结果表明，BAMAS 在取得可比性能的同时，最多可将成本降低 86%。

## 创新点

1. **将预算约束显式纳入多智能体系统设计**：现有框架如 AutoGen 和 MetaGPT 主要采用反应式策略，将成本视为事后考量。BAMAS 首次将多智能体系统的构建问题形式化为在固定预算下的组合优化问题，使得成本成为一等约束而非副产品。

2. **整数线性规划驱动的模型配备策略**：论文提出性能优先的层级化选模策略，将可用大语言模型按能力划分为多个层级，并通过整数线性规划在预算约束下最大化总体性能权重。这一策略避免了弱模型盲目堆叠导致的成本浪费，同时确保高能力模型被优先使用。

3. **离线强化学习驱动的协作拓扑选择**：不同于固定拓扑的工作流设计，BAMAS 训练一个策略网络，根据任务描述和预算上限从线性、星型、反馈和规划器驱动四种拓扑中选择最优协作模式。训练采用带熵正则的 REINFORCE 算法，在离线经验数据集上优化，避免了在线试错的高昂调用开销。

4. **可调的预算-性能权衡机制**：BAMAS 的成本具有可预测性和可调性。实验表明，增加预算可直接转化为更高的任务准确率，而系统在绝大多数任务中均能将实际花费控制在预算以内，这使其更适合生产环境的资源规划。

## 一句话总结

BAMAS 通过整数线性规划选择预算内最优的大语言模型集合，再利用离线强化学习为其匹配最佳协作拓扑，从而在三个推理基准上实现了与现有方法相当甚至更高的准确率，同时将成本降低最高达 86%。

## 研究问题

论文针对的核心问题是：给定任务描述、可用大语言模型池和成本预算，如何构建一个多智能体系统，使其在严格满足预算约束的前提下实现最优的性能-成本权衡。具体而言，需要同时确定两个互相关联的决策变量：

- 选择哪些大语言模型作为系统内的智能体，即模型配备问题；
- 这些智能体之间应采用何种交互拓扑结构，即协作模式问题。

这一问题的动机来源于当前多智能体系统在实际部署中的成本困境。单次任务可能涉及数十次大语言模型调用，而成本随协作拓扑和推理深度不可预测地增长。现有框架通常以最大化性能为首要目标，对成本缺乏前置控制，导致系统难以在真实业务场景中可靠扩展。

## 数据与任务定义

**评估数据集**

论文在以下三个广泛使用的基准数据集上进行评估：

- **GSM8K**：小学数学文字题数据集，共 1319 道测试题，用于评估数学推理能力。
- **MBPP**： Mostly Basic Python Programming 数据集，包含 500 道入门级 Python 编程题，用于评估代码生成能力。
- **MATH**：高中竞赛级数学问题数据集。由于规模较大，论文按难度等级和题目类型进行分层抽样，从中构建 1000 道题的训练集和测试集，保持原始分布不变。

**评估指标**

- **准确率**：任务成功率，即多智能体系统输出正确答案的任务占比。
- **平均成本**：所有任务的平均调用成本。单次调用的成本计算公式如下：

$$c_i = T_{\text{in}} \cdot P_{\text{in}} + T_{\text{out}} \cdot P_{\text{out}}$$

其中下标 in 与 out 分别对应输入与输出，乘数项为对应 token 数与模型厂商单价的积。为便于展示，结果乘以 $10^6$ 进行缩放。

- **超预算任务数**：实际花费超过给定预算的任务数量。

**基线方法**

- **AutoGen**：微软开发的多智能体对话框架，采用对话式协作模式。
- **MetaGPT**：基于角色扮演和标准化流程的多智能体框架。
- **Naive-CostAware**：论文构建的简化基线，仅做成本感知的模型配备，不做拓扑优化，用于验证联合优化的必要性。

## 方法主线

BAMAS 的整体架构包含三个串行组件：预算约束下的大语言模型配备、智能体协作拓扑选择、以及系统实例化与执行。

### 预算约束下的大语言模型配备

该组件从可用模型池中选出子集，使得总成本不超过预算。作者基于以下观察采用性能优先策略：对于复杂推理任务，单个高性能模型通常优于多个弱模型的简单组合。因此，优化目标是在预算内最大化所选模型的总体性能权重。

具体实现上，所有可用模型按 LMSys Chatbot Arena 排行榜排名划分为多个层级，最高层级代表最强性能，最低层级代表最弱性能。为量化性能差异，设最低层级权重为 1，更高层级的权重递归计算如下：

$$W_i = 1 + \sum_{j=i+1}^{L} \left\lfloor W_j \cdot \frac{B}{c_j} \right\rfloor$$

其中 $c_j$ 为第 $j$ 层级模型的单次调用成本估算。该设计确保更高性能层级的模型获得显著更大的权重，从而在整数线性规划求解中被优先选择。

由于实际执行前无法获知精确 token 用量，论文采用代表性估算：输入 token 数固定为 500（基于先前测量，典型输入长度在 128 至 256 之间，论文将上限放宽至 500 以容纳多智能体场景中的额外上下文）；输出 token 数则从目标数据集中采样 50 条训练实例，取观察到的最大输出长度作为上限。

引入二元决策变量表示是否选择第 $i$ 层级的第 $j$ 个模型实例，得到如下整数线性规划：

$$\max \sum_{i=1}^{L} \sum_{j} W_i \cdot x_{ij}$$

$$\text{s.t.} \quad \sum_{i=1}^{L} \sum_{j} c_i \cdot x_{ij} \leq B, \quad \sum_{i=1}^{L} \sum_{j} x_{ij} \geq 2$$

第二个约束确保系统至少包含两个智能体，以形成有意义的多智能体协作。求解该规划即可得到预算可行的候选模型池。

### 智能体协作拓扑选择

在确定模型池后，系统需要选择具体的协作拓扑。BAMAS 定义了四种离散拓扑模式：

- **线性拓扑**：智能体按固定顺序链式传递信息，适用于结构化分解任务。
- **星型拓扑**：一个中心智能体协调多个边缘智能体，适合需要集中决策的场景。
- **反馈拓扑**：智能体之间形成循环反馈链路，支持迭代优化。
- **规划器驱动拓扑**：权重最高的模型担任规划器，次高权重模型担任批评者，其余模型作为执行者，适用于开放式任务。

拓扑选择由策略网络完成，以任务描述和预算为条件。训练采用离线强化学习，在预先收集的经验数据集上进行，目标为最大化复合奖励。训练损失函数如下：

$$\mathcal{L}(\theta) = -\hat{\mathbb{E}} \left[ \log \pi_\theta(t \mid T, B) \cdot R_{\text{final}}(\tau) \right] - \beta \cdot H(\pi_\theta(\cdot \mid T, B))$$

其中 $R_{\text{final}}(\tau)$ 为轨迹的最终复合奖励，包含任务成功奖励（任务成功时加分，失败时减分）和效率奖励（对超预算给予强惩罚、对节省预算给予适度奖励）。$H(\cdot)$ 为策略熵，$\beta$ 为熵正则系数。训练过程中，保留在整个离线数据集上取得最高平均复合奖励的参数作为最终策略。

### 系统实例化与执行

根据整数线性规划求解得到的模型池和策略网络选定的拓扑，BAMAS 实例化对应的多智能体系统并执行任务。各智能体按照选定拓扑的通信规则进行交互，最终输出任务结果。整个流程从任务输入到系统输出形成完整的端到端执行链。

### 机制流程

BAMAS 的核心执行链可分为以下四个步骤：

1. **输入任务与预算，执行预算约束模型配备**
   - 输入：任务描述、可用模型池、成本预算。
   - 操作：按排行榜对模型池进行层级编码，估算各模型调用成本，建立并求解整数线性规划以最大化性能权重和，提取预算可行的候选模型子集。
   - 输出去向：得到候选模型子集。

2. **输入模型子集与任务信息，查询并选择协作拓扑**
   - 输入：候选模型池、任务描述、预算。
   - 操作：将任务描述和预算编码为条件向量，查询训练好的拓扑选择策略网络，从四种候选拓扑中解码并采样得到最优拓扑。
   - 输出去向：得到最优协作拓扑。

3. **输入模型子集与拓扑，实例化多智能体系统**
   - 输入：模型子集和协作拓扑。
   - 操作：为每个选中的大语言模型实例化一个智能体角色（规划器、批评者或执行者），并按照拓扑建立通信通道，拼接形成可执行的完整系统。
   - 输出去向：得到可执行的完整多智能体系统实例。

4. **输入具体任务实例，执行并产出结果**
   - 输入：待解决的具体任务实例。
   - 操作：各智能体按拓扑规则进行多轮推理、通信和协作，更新中间状态，直至产生最终答案或达到终止条件。
   - 输出去向：任务执行结果及其实际调用成本。

## 关键结果

**成本-性能权衡**

表一和表二报告了 BAMAS 与基线方法在三个数据集上的平均成本和准确率。BAMAS 在所有数据集上均展现出优于现有方法的成本-性能权衡。

- **GSM8K**（预算 1625）：BAMAS 达到 95.3% 的准确率，几乎追平 AutoGen 配合 DeepSeek-V3 的 95.4%，但平均成本仅为 542.9，相比基线的 1425.3 降低 62%。
- **MBPP**（预算 1625）：BAMAS 准确率为 82.6%，与 SOTA 的 82.2% 相当，但平均成本仅 529.2，相比基线的 3735.1 降低 86%。
- **MATH**（预算 2000）：BAMAS 准确率达到 81.2%，超过现有最佳方法的 77.6%，同时保持更低的平均成本 646.0。

图二进一步以二维散点图展示了这一趋势：BAMAS 在不同预算设置下形成的点簇位于成本-准确率平面的左上方，即相同成本下准确率更高、相同准确率下成本更低。重要的是，BAMAS 具有可调性，增加预算可直接转化为更高的准确率，这为实际部署中的资源规划提供了明确指导。

**预算遵守情况**

表三报告了超预算任务数量。在 GSM8K 的 1319 个任务中，BAMAS 未出现任何超预算情况；在 MBPP 的 500 个任务中，超预算数量在 1 至 5 个之间；在 MATH 的 1000 个任务中，最高超预算数为 30（预算 2000 时），仅占 3%。总体而言，BAMAS 的预算违反率极低。

**联合优化的必要性**

表四对比了 BAMAS 与 Naive-CostAware 基线。在 GSM8K 上，BAMAS 以 542.9 的成本达到 95.3% 的峰值准确率，而该基线达到相同准确率需要 1650.8 的成本。在 MBPP 上，BAMAS 不仅准确率更高（82.6% 对 81.6%），成本也更低（529.2 对 1379.1）。这证明仅做成本感知的模型配备而不优化协作拓扑是不够的，两者的联合优化对实现最优权衡至关重要。

## 深度分析

**整数线性规划权重设计的工程意义**

BAMAS 的整数线性规划组件中最具巧思的是层级权重的递归定义。通过让更高层级的模型权重不仅线性增长，还与其成本和预算的比值相关联，系统能够在预算紧缩时自动滑落到性价比更优的次高模型组合，而非刚性追求顶级模型。这种设计使得同一套整数线性规划框架可以适应从 500 到 3000 不等的多种预算档位，而无需手工调整阈值。

然而，这一权重公式的一个隐含假设是 LMSys Arena 排名能够可靠地代理下游任务性能。对于代码生成和数学推理这类特定领域任务，通用聊天排行榜的排名是否完全适用仍值得商榷。论文未在消融实验中验证更换性能代理对最终结果的影响。

**离线强化学习的训练效率与策略稳定性**

拓扑选择策略采用离线 REINFORCE 训练，这避免了在线与环境交互带来的高昂接口调用开销，是该方法在生产场景中可行的关键。训练使用 20000 的批次大小和仅 10 轮迭代，表明策略学习的问题规模相对可控。但离线强化学习的一个固有风险是分布偏移：策略可能在训练集上表现良好，但对未见过的任务类型或预算档位泛化不足。论文未明确报告策略在跨数据集或跨预算泛化测试中的性能波动。

**静态 token 估算的误差传递**

BAMAS 对输入 token 采用固定估算（500），对输出 token 采用训练集采样上限（从 50 个样本估计）。这种静态估算在生产环境中存在系统误差风险：当实际任务输入显著超过 500 token，或模型在多轮交互中生成远超训练样本长度的输出时，整数线性规划的成本约束将失效，导致超预算。表三中 MATH 数据集 3% 的超预算率可能正源于此——竞赛级数学问题的中间推理步骤往往长于小学数学问题。

**拓扑空间的限制**

BAMAS 将拓扑限定为四种离散模板，虽然覆盖了主流协作模式，但排除了更复杂的动态拓扑（如执行过程中根据中间结果增删智能体、或自适应调整通信图结构）。对于需要深度多步拆解的开放式任务，固定拓扑可能限制系统的表达能力。这也是规划器驱动拓扑被引入的原因，但它仍然依赖于预定义的角色分配规则，而非完全动态的组织结构。

## 局限

1. **离线训练数据的覆盖范围有限**：拓扑选择策略依赖预先收集的离线经验数据集。论文未详细说明该数据集的规模、收集方式以及是否覆盖足够的任务-预算组合。如果数据在特定预算区间或任务类型上稀疏，策略的泛化能力将受限。

2. **token 用量的前置估算存在系统偏差**：输入和输出 token 数均为执行前的静态估计，未考虑任务实例间的实际差异。对于输入长度分布高度偏斜的数据集，固定估算值 500 可能导致大量任务的成本估算偏离实际，进而影响整数线性规划解的最优性和预算遵守率。

3. **拓扑候选集规模较小**：仅有四种离散拓扑可选，虽然通过强化学习实现了自适应选择，但候选空间本身限制了系统可能达到的性能上限。更细粒度的拓扑变体（如不同深度的反馈链、不同数量的星型分支）未被纳入考虑。

4. **缺乏对推理时扩展性的讨论**：论文聚焦于单任务的成本控制，未讨论当系统需要处理高并发任务流时的资源调度问题。在生产环境中，预算约束通常是在时间窗口内对多个任务的总体限制，而非每个任务的独立限制。

5. **实验未涵盖开放域对话或工具使用场景**：评估集中在数学推理和代码生成这两个具有明确答案的封闭任务上。BAMAS 在开放域多轮对话、需要调用外部工具的复杂任务上的表现尚不明确。

## 我的笔记

BAMAS 的核心洞察是将多智能体系统的构建问题从先设计协作流程再尽量控制成本转变为在预算约束下联合优化模型配备和协作结构。这一视角转换与机器学习系统中的模型路由和专家混合思想有共通之处：都是在有限计算预算下做动态资源分配。不同之处在于 BAMAS 的决策粒度更粗（模型级别而非层级别），且额外引入了拓扑维度。

从复现角度，BAMAS 的工程实现相对轻量：整数线性规划部分可用任何标准求解器实现，强化学习部分则是标准的 REINFORCE 加熵正则。最具挑战的环节可能是离线经验数据集的构建——需要为大量任务、预算、拓扑组合运行完整的多智能体执行流程以收集奖励信号。对于没有充足接口预算的研究团队，这一步可能成为复现瓶颈。

一个值得延伸的方向是将 BAMAS 与在线学习结合：先用离线数据训练初始策略，部署后持续收集实际执行反馈，通过保守的在线更新逐步改进拓扑选择。另一个方向是放宽静态拓扑假设，允许智能体在执行过程中根据中间结果动态重组通信结构，这将使系统更贴近真实团队的问题解决模式。
