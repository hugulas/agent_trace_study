# AgentPro: Enhancing LLM Agents with Automated Process Supervision

## 核心信息

- 标题: AgentPro: Enhancing LLM Agents with Automated Process Supervision
- 作者: Yuchen Deng, Shichen Fan, Naibo Wang, Xinkui Zhao, See-Kiong Ng
- 年份: 2025
- 会议 / 期刊: Proceedings of the 2025 Conference on Empirical Methods in Natural Language Processing (EMNLP 2025)

## 原文摘要翻译

大语言模型智能体通过思维链推理和工具调用等机制，在解决复杂任务方面展现出巨大潜力。然而，现有框架在推理过程中缺乏显式监督，这可能导致错误在推理链中传播，并阻碍中间决策阶段的优化。本文提出了一种新颖的框架 AgentPro，通过自动化的过程监督来提升大语言模型智能体的性能。AgentPro 采用蒙特卡洛树搜索自动生成分步级别的标注，并基于这些标注训练过程奖励模型，从而实现对推理过程的细粒度质量评估。通过采用拒绝采样策略，大语言模型智能体动态调整生成概率分布，以防止错误路径的延续，进而提升推理能力。在四个数据集上的大量实验表明，本文方法显著优于现有的基于智能体的大语言模型方法（例如在 HotpotQA 数据集上准确率提升了 6.32%），这凸显了其在处理复杂推理链方面的优势。

## 创新点

1. **首个将自动化过程监督整合进 LLM 智能体推理与决策的框架**。传统方法如 ReAct、Reflexion 等缺乏对中间步骤的显式验证机制，而 AgentPro 通过过程奖励模型在每一步进行实时质量评估，有效缓解了错误传播问题。
2. **基于蒙特卡洛树搜索的自动标注算法**。为避免人工逐步骤标注的高昂成本，作者利用 MCTS 从当前步骤出发模拟多条后续路径，根据最终答案的正确性自动生成正负标签，从而以自监督方式构建 PRM 训练数据。
3. **拒绝采样与全参数微调的强化学习训练策略**。在 PRM 训练完成后，AgentPro 通过迭代生成多个候选响应、由 PRM 评分并选出最优响应进行监督微调，逐步提升智能体的推理能力。
4. **跨任务域的广泛验证**。实验覆盖事实验证、多跳问答、具身决策和在线购物决策四类场景，系统性地验证了过程监督在推理任务与决策任务中的通用增益。

## 一句话总结

AgentPro 利用蒙特卡洛树搜索自动构造步骤级训练标签来训练过程奖励模型，再通过拒绝采样策略筛选高分响应并微调智能体，从而在四个基准数据集上显著提升了大语言模型智能体的推理与决策准确率。

## 研究问题

当前的大语言模型智能体框架（如 ReAct、Reflexion、ExpeL）在处理多步复杂任务时，通常只对最终输出进行验证，而缺乏对中间推理步骤的显式监督。这种缺陷会导致以下问题：

- **错误传播**：一旦某个中间步骤出错，后续步骤会在错误基础上继续推导，最终导致整体失败。
- **难以优化**：没有细粒度的步骤级反馈，智能体无法针对性地改进其决策策略。
- **标注成本高**：过程奖励模型的训练需要大量的人工步骤级标注，这在实际应用中成本极高。

因此，本文的核心研究问题是：如何以自动化的方式生成可靠的步骤级监督信号，并据此训练过程奖励模型，从而有效提升 LLM 智能体在多步推理与决策任务中的表现？

## 数据与任务定义

本文在四个具有代表性的基准数据集上评估了 AgentPro，覆盖了知识推理与交互决策两类任务：

- **FEVER**（Thorne et al., 2018）：事实抽取与验证任务，要求模型判断给定声明是否被维基百科内容支持、反驳或信息不足。
- **HotpotQA**（Yang et al., 2018）：知识密集型的多跳问答任务，需要模型跨多个文档进行推理以回答问题。
- **ALFWorld**（Shridhar et al., 2020）：具身模拟环境中的多步决策任务，要求智能体在家庭场景中完成诸如寻找物品、操作电器等指令。
- **WebShop**（Yao et al., 2022a）：模拟在线购物的复杂决策任务，涉及多轮交互、属性筛选与商品匹配。

评估指标统一采用成功率：FEVER 与 HotpotQA 使用精确匹配，ALFWorld 使用按时任务完成率，WebShop 使用成功购买匹配全部属性的比例。

基线方法包括 Act、ReAct、Reflexion 和 ExpeL。所有方法共享相同的基础模型。在涉及百科知识的数据集上，问题后附加维基百科上下文作为额外信息。实验以 LLaMA-3.1-8B-Instruct 为骨干模型。

## 方法主线

AgentPro 的核心思路是将过程监督自动化的三个环节串联起来：自动标注、PRM 训练、拒绝采样微调。整个框架围绕两个核心模型展开，分别记为 $M_a$（待优化的智能体）与 $M_p$（过程奖励模型）。

### 机制流程

1. **输入查询并生成逐步轨迹**。给定查询 $q$，智能体 $M_a$ 生成包含 $K$ 个步骤的解答序列，其中每一步在决策任务中可表示为动作与观测组成的二元组。完整轨迹记为 $T_q$，作为后续自动标注的输入。
2. **蒙特卡洛树搜索自动标注**。对于轨迹中的每个中间步骤，以当前前缀为起点，利用智能体模拟 $m$ 条后续路径。若任意路径的最终答案与标准答案一致，则该步骤标记为正；否则标记为负。该标注规则可形式化为：
$$\text{Label}(s_i) = \begin{cases} + & \text{if } \exists j \in [1, m], A_{\text{final}}^{(j)} = A_{\text{correct}} \\ - & \text{if } \forall j \in [1, m], A_{\text{final}}^{(j)} \neq A_{\text{correct}} \end{cases}$$
   所有查询标注完成后，构造数据集 $D_{\text{prm}}$ 用于训练 PRM。
3. **训练过程奖励模型**。利用自动标注数据集 $D_{\text{prm}}$，对 $M_{\text{prm}}$ 进行全参数微调，目标是最小化交叉熵损失，使其能够准确预测每个步骤 $s_i$ 的正负概率。训练后的 PRM 可为任意推理步骤输出质量分数，从而具备评估步骤级有效性的能力。
4. **拒绝采样微调智能体**。在每一轮训练中，针对每个查询 $q$，$M_{\text{agent}}$ 生成 $n$ 个候选响应。PRM 对每个响应的每一步给出标签 $l_i$ 及概率 $P_i$，响应的整体奖励分数 $R_j$ 计算为各步骤概率的带符号平均：
$$R_j = \frac{1}{K} \sum_{i=1}^{K} \bigl( \mathbb{I}(l_i = +) \cdot P_i - \mathbb{I}(l_i = -) \cdot P_i \bigr)$$
   选取奖励最高的响应 $r^* = \arg\max_{r_j} R_j$ 作为该查询的最优样本，随后对 $M_{\text{agent}}$ 进行监督微调以最大化复现最优响应的似然。该过程迭代 $T$ 轮，逐步提升智能体生成高质量推理链的能力。

### PRM 的训练目标

过程奖励模型的训练采用标准交叉熵损失。对于第 $j$ 个训练样本中包含的 $K$ 个步骤，损失函数为：
$$L_{\text{prm}} = -\frac{1}{N} \sum_{j=1}^{N} \frac{1}{K} \sum_{i=1}^{K} \text{CrossEntropy}\bigl(y_i^{s_j}, \hat{y}_i^{s_j}\bigr)$$
其中 $\hat{y}_i^{s_j}$ 表示 PRM 对第 $j$ 个样本中第 $i$ 步的预测概率。全参数微调策略确保 PRM 能够充分利用细粒度监督信号，准确判断每个生成步骤是否逻辑上朝正确最终答案推进。

### 智能体的监督微调目标

在拒绝采样阶段，选取所有查询对应的最优响应后，对智能体 $M_{\text{agent}}$ 进行全参数监督微调，损失函数为：
$$L_{\text{agent}} = -\frac{1}{N_r} \sum_{i=1}^{N_r} \sum_{t=1}^{T_i} \log P(y_{i,t} \mid x_i)$$
其中 $x_i$ 为第 $i$ 个训练提示，$y_i$ 为对应的最优响应序列，$T_i$ 为其长度。该目标促使智能体在后续生成中更倾向于输出被 PRM 高度评价的推理路径。

## 关键结果

### 主实验结果

在 LLaMA-3.1-8B-Instruct 上的主实验表明，AgentPro 在四个数据集上均稳定超越所有基线：

- **FEVER**：相比最优基线 ExpeL（51.87%）提升 3.59%，达到 55.46%（按文中 Fig. 3 及描述推算）。
- **HotpotQA**：相比 ExpeL 提升 6.32%，验证了过程监督在多跳推理链中的显著增益。
- **ALFWorld**：在 134 个家庭交互场景中达到 40.88% 的成功率，超过 Reflexion（34.08%），说明自动生成的步骤级标注成功捕捉了环境状态转移。
- **WebShop**：成功率达到 52.67%，大幅领先 ExpeL（38.00%）和 Reflexion（40.67%），表明 PRM 对动作质量（如价格比较、属性匹配）的评估能力有效帮助智能体构建更优的决策链。

## 深度分析

AgentPro 的设计将过程监督从数学推理领域扩展到了更广泛的智能体任务，包括事实验证、多跳问答和交互式决策。其最核心的工程洞察在于：不需要昂贵的人工标注，仅利用智能体自身的生成能力（通过 MCTS 模拟）就可以构造足够质量的步骤级训练信号。

从训练范式来看，AgentPro 实际上构建了一个自举提升的闭环：智能体生成轨迹 → MCTS 自动标注 → PRM 学习评估 → 拒绝采样筛选最优轨迹 → 智能体微调以复现最优轨迹。这个循环与 RLHF 中的奖励模型训练思路一致，但将监督粒度从整条响应下沉到了单个步骤，并将标注成本从人工转移到了自动模拟。

然而，该框架也存在一些值得深入思考的工程权衡。首先，蒙特卡洛树搜索标注的正确性依赖于智能体自身完成后续步骤的能力。如果基础模型在特定任务上表现极差，那么从错误步骤出发的模拟路径很可能全部失败，导致该步骤被错误地标记为负标签（假阴性），或者正确步骤因模拟不足而被误判。在消融实验中，单次模拟的准确率仅为 32%，三次模拟跃升至 39%，暗示了模拟次数对标注质量的关键影响。其次，全参数微调奖励模型和智能体带来了较高的训练成本。论文中提到实验使用单台配备英伟达高端显卡的机器，虽然比人工标注便宜，但相比纯提示工程方法仍有显著的计算开销。

此外，PRM 的奖励分数设计采用了简单的带符号平均概率。这种方法对长序列中的局部错误敏感，但未显式考虑步骤之间的时序依赖或因果结构。在某些任务中，一个早期步骤可能为后续多个关键动作奠定基础，而平均化评分可能无法充分反映这种结构性贡献。未来的改进方向可以探索更精细的奖励聚合策略，例如引入折扣因子或基于注意力机制的步骤重要性加权。

## 局限

作者在论文末尾明确指出了以下局限：

1. **训练成本高**：与基于提示的方法相比，AgentPro 需要对大语言模型进行额外训练以开发过程奖励模型并增强智能体。具体而言，MCTS 生成中间步骤标签以及全参数微调均消耗大量计算资源和时间。
2. **对基础模型能力的依赖**：MCTS 自动标注的有效性受限于智能体本身完成模拟路径的能力。若基础模型在某一领域的推理能力较弱，自动标注的质量可能下降。
3. **未探索参数高效微调**：当前实现采用全参数微调策略，作者指出未来可以探索参数高效微调（如 LoRA）和蒸馏技术，以进一步降低资源消耗。

值得补充的是，论文未在实验中对 PRM 标注错误率进行直接量化，也未深入分析 MCTS 标注中假阴性与假阳性的比例及其对最终性能的影响。此外，四个数据集的实验均采用统一的 $m=3$、$n=4$、$T=5$ 配置，未针对不同任务特性进行超参数自适应调整，这可能存在优化空间。

## 我的笔记

AgentPro 的自动化过程监督思路对构建可复现的智能体训练 pipeline 具有直接参考价值。其核心优势在于将标注成本从人工转移到计算，这对于需要大量步骤级监督的真实应用场景尤为重要。在实际复现时，有几个关键点需要特别注意：

- **MCTS 模拟的实现细节**：论文中每个步骤的模拟需要从前缀上下文继续生成完整路径直至终止条件。在具身环境（如 ALFWorld）中，这意味着需要与环境进行多轮交互，模拟开销可能显著高于纯文本推理任务。实现时应考虑并行化模拟以控制 wall-clock 时间。
- **PRM 的输入表示**：PRM 需要对步骤 $s_i$ 及其上下文进行编码并输出二分类概率。实现上可以将前缀文本拼接后输入因果语言模型，在对应步骤位置取 hidden state 并投影为分类 logits。
- **拒绝采样的迭代稳定性**：迭代训练 $T=5$ 轮可能不足以使智能体收敛到稳定策略，特别是在复杂决策任务中。复现时可尝试增加迭代轮数或引入早停机制，监控验证集上的成功率变化。
- **与在线 RL 的对比**：AgentPro 采用离线拒绝采样而非在线 PPO，这降低了训练不稳定性，但也限制了智能体探索新策略的能力。对于需要强探索的任务，可以考虑将 PRM 与在线 RL 结合。

从研究脉络来看，AgentPro 与 Math-Shepherd、ReST-MCTS 等工作共享"利用搜索自动生成过程监督信号"的思想，但将其从数学领域推广到了更通用的智能体任务。这一方向值得关注：随着 LLM 智能体在工具调用、网页浏览、代码执行等场景中的应用加深，自动化过程监督将成为提升可靠性的关键技术路径。
