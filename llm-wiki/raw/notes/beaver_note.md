# BEAVER: A Training-Free Hierarchical Prompt Compression Method via Structure-Aware Page Selection

## 原文摘要翻译

大语言模型上下文窗口的指数级扩展虽然释放了长文档理解的能力，但也带来了推理延迟和信息利用方面的严重瓶颈。现有的压缩方法往往因激进的词元剪枝而面临高昂的训练成本或语义碎片化问题。本文提出 BEAVER，一种全新的无需训练框架，将压缩方式从线性词元移除转向结构感知的层次化选择。BEAVER 通过双路池化将变长上下文映射为密集的页级张量，从而最大化硬件并行度；并通过结合语义与词汇双分支选择以及句子平滑的混合规划器来保持语篇完整性。在四个长上下文基准上的广泛评估表明，BEAVER 取得了与 LongLLMLingua 等当前最优方法相媲美的性能。特别是在 RULER 基准上，BEAVER 在多针检索任务中保持了高保真度，而基线方法则出现明显退化。在效率方面，BEAVER 在 128k 上下文上将延迟降低 26.4 倍，为高吞吐量应用提供了可扩展的解决方案。代码见 https://cslikai.cn/BEAVER/。

## 创新点

1. **结构感知的层次化页面选择**：将压缩粒度从词元提升到语义完整的页面，避免碎片化，同时使变长序列转化为 GPU 友好的固定形状张量。
2. **双路池化页面编码器**：同时通过均值池化捕获全局语义、最大池化保留词汇异常值，并引入 ITF 权重实现上下文自适应调整，无需外部训练。
3. **三重结构先验查询规划器**：整合锚点页、流式页与闪存页先验，模拟人类阅读习惯，在压缩过程中抑制语义漂移。
4. **无需训练的即插即用设计**：不依赖数据蒸馏或强化学习，跨模型规模（0.6B–32B）与不同任务均具备良好泛化性。
5. **显著的效率与性能平衡**：在 128k 上下文上实现 26.4 倍延迟降低，同时在问答与细粒度检索任务上达到或超越当前最优监督方法。

## 一句话总结

BEAVER 是一种无需训练的层次化提示压缩框架，通过结构感知的页面选择与双路池化编码，在保持语义连贯的同时将长上下文推理延迟降低 26.4 倍，并在多个长文本基准上达到或超越现有最优方法。

## 研究问题

大语言模型的上下文窗口已从早期的 32k 扩展到百万词元级别，但这一扩展带来了两个核心瓶颈。

第一，**计算墙**。自注意力的计算复杂度为 $O(L^2)$，导致预填充延迟随上下文长度激增，造成首词元时间与尾延迟过长。尽管量化与系统级优化可缓解内存压力，但注意力本身的二次复杂度仍是根本瓶颈。

第二，**信息利用的边际递减**。单纯扩展上下文长度并未带来线性收益，反而触发中间信息丢失效应，即模型倾向于忽略中间位置的关键信息。

现有提示压缩方法主要分为两类：无监督统计方法（如 Selective-Context、LongLLMLingua）依赖困惑度或自信息指标，虽无需训练但受限于启发式假设；监督学习方法（如 LLMLingua-2、PCRL）将压缩视为分类或排序任务，精度更高但训练成本大且跨模型泛化差。此外，现有方法多采用非结构化的词元级剪枝，容易破坏语义与句法连贯性，导致碎片化上下文。因此，如何在无需训练的前提下实现结构保留且硬件高效的压缩，是本论文要解决的核心问题。

## 数据与任务定义

### 基准数据集

- **LongBench**：涵盖单文档问答、多文档问答、摘要、小样本学习等 6 大类任务，平均长度 3.6k–18.4k 词元，评估指标为 F1 与 Rouge-L。
- **ZeroSCROLLS**：包含 10 个子任务，覆盖摘要、问答等长上下文场景，评估指标包括 Rouge-L、F1 等。
- **RULER**：高质量合成基准，上下文长度覆盖 4k–128k，任务包括单针检索、多针检索、变量追踪等，评估指标为准确率。
- **L-Eval**：包含 20 个数据集，强制严格词元长度约束，用于评估域外泛化能力，评估指标为 F1 与准确率。

### 基线方法

- 无监督统计方法：Selective-Context、LongLLMLingua。
- 监督学习方法：LLMLingua、LLMLingua-2。
- 嵌入检索方法：SBERT、OpenAI Embeddings。

### 实验设置

- 下游推理后端采用 GPT-3.5-turbo-instruct 模型。
- BEAVER 的页面编码器使用 Qwen3-8B 提供嵌入。
- 关键超参数：页面大小 $M=64$，融合权重 $\gamma=0.7$，评分参数 $\lambda=0.7$。

## 方法主线

### 总体架构

BEAVER 包含三个核心组件：Segmenter（分段器）、PageEncoder（页面编码器）与 QueryPlanner（查询规划器）。整体思路是先将变长提示映射为固定大小的二维页张量，再通过双路池化获得层次化页面向量，最后结合查询相关性与结构先验选择保留页面，并经句子平滑恢复为连贯文本。

### 机制流程

1. **分段与分页（Segmenter）**。输入长上下文序列 $C$ 与查询 $Q$。

    分段器依据语义分隔符将上下文切分为逻辑段。随后填充为二维页面张量 $P$，其形状为 $N$ 行 $M$ 列，页面大小 $M=64$。

    若查询显式出现，则将其起始页面标记为查询锚点 $p_{\text{qry}}$，压缩仅作用于查询之前的上下文，并施加因果约束。

    若查询隐式位于序列末尾，则通过后向窗口定位自然语言边界以避免截断。

2. **双路池化编码（PageEncoder）**。对页面张量中的每个词元计算逆词频权重，以突出罕见但关键的词元。随后，对每页分别执行加权平均池化与最大池化，再通过融合权重组合为最终页面向量。

加权平均池化：

$$\mu_i = \frac{\sum_{j=1}^{M} \omega_{i,j} x_{i,j}}{\sum_{j=1}^{M} \omega_{i,j} + \varepsilon}$$

最大池化：

$$m_i = \max_{1 \le j \le M} \big[ B_{i,j} x_{i,j} + (1 - B_{i,j}) \cdot \beta \big]$$

融合：

$$p_i = \gamma \mu_i + (1 - \gamma) m_i$$

其中 $B_{i,j}$ 为有效词元掩码，$\varepsilon$ 保证数值稳定，$\beta$ 为填充位置偏置，实验中 $\gamma=0.7$。

3. **混合查询规划（QueryPlanner）**。将查询分解为子查询向量，计算其与各页面向量的混合语义-词汇相关性得分。查询规划器引入三重结构先验：锚点（保留查询附近页面）、流式（保留局部连续页面）与闪存（保留全局高频显著页面）。基于这些先验构建显著性掩码，筛选高得分页面。

评分公式如下：

$$s_{i,k} = w_{\text{itf}}(q_k) \cdot \frac{\hat{p}_i \cdot q_k}{\|\hat{p}_i\|_2 \cdot \|q_k\|_2}$$

4. **句子平滑与输出（Sentence Smoother）**。将选中的页面边界向外扩展至最近的句边界，恢复语法完整的连续子序列 $\tilde{C}$，作为压缩后的提示输入下游 LLM。该机制避免页面截断导致的句法断裂，保证输出可读性。

### 关键设计细节

- **ITF 权重自适应**：基于上下文词元频率统计动态调整权重，无需外部训练数据即可突出关键实体与数字。
- **GPU 友好的张量操作**：将不规则的变长序列转换为固定形状的二维页张量，充分利用 GPU 并行计算，避免动态长度带来的内核启动开销。
- **结构先验的可解释性**：锚点先验模拟“问题附近找答案”的人类阅读习惯；流式先验保持局部语篇连贯；闪存先验捕捉全局高频信息，三者互补抑制语义漂移。

## 关键结果

### 主基准性能

在 LongBench 与 ZeroSCROLLS 基准上，BEAVER 在 2,000 与 3,000 词元预算约束下整体优于 LLMLingua-2 与 LongLLMLingua。

以 LongBench 单文档问答为例，BEAVER 在多项子任务上取得最优或次优成绩，平均压缩后词元数约为 1,900–2,100，压缩比约 $1/\tau \approx 4\times$。

## 深度分析

### 无需训练的优势与边界

BEAVER 的无需训练设计是其最大差异化因素。通过利用上下文内 ITF 统计与结构先验，它避免了数据蒸馏或强化学习所需的大量标注数据与计算开销。实验显示，在 Qwen3-0.6B 上，学习型基线 LLMLingua 已出现严重性能崩溃（单针 5.0%，多针 4.0%），而 BEAVER 仍保持 100% 单针准确率与 95% 多针准确率。这说明无需训练的方法在小模型或资源受限场景下具有独特优势。

### 页级粒度：效率与精度的权衡

将压缩粒度从词元提升到页面（$M=64$）是 BEAVER 效率提升的关键。

它使变长序列转化为固定大小的二维张量，便于 GPU 批量并行。然而，页级粒度天然不如词元级精细，可能保留选中段内的轻微冗余。

论文消融实验显示，将页面大小从 $M=64$ 改为 $M=32$ 或 $M=128$ 分别导致 LongBench QA 平均分下降 1.8 与 2.4 分，说明粒度选择对精度有直接影响。

### 双路池化的互补作用

PageEncoder 的均值池化捕获页面全局语义主题，最大池化则保留词汇层面的显著信号（如数字、专有名词）。消融实验表明，移除最大池化导致平均分下降 2.7 分，移除均值池化下降 2.6 分，两者贡献相当。融合权重 $\gamma=0.7$ 偏向均值池化，暗示全局语义在多数任务中略占主导，但最大池化对细节敏感任务（如 RULER 的变量追踪）不可或缺。

### 三重结构先验的机制解读

QueryPlanner 的锚点、流式、闪存先验可被视为对“人类如何浏览长文档”的显式建模。锚点强制保留查询附近的页面，防止截断导致的答案丢失；流式保留局部连续页面，维持语篇的因果与逻辑链条；闪存通过全局高频词分布捕捉跨段重复出现的主题信息。消融结果显示，单独移除锚点导致性能暴跌 21.7 分，说明查询附近上下文对问答任务至关重要；移除闪存仅下降 2.7 分，表明其在多数场景中为辅助角色。

### 相似度检索的隐含瓶颈

BEAVER 的检索机制本质基于语义与词汇相似度。这在证据与查询存在直接词汇重叠时表现优异，但在需要深度多跳推理的场景中，中间证据可能与查询仅有间接关联，相似度得分难以有效捕捉。论文亦指出该局限性，提示未来可探索基于推理链的显式中间节点扩展。

## 局限

1. **页级冗余**：页面粒度优先保证结构完整与速度，但可能保留段内次要冗余信息，无法像词元级剪枝那样精确到单个词元。
2. **多跳推理挑战**：依赖语义-词汇相似度的检索机制在深层多跳推理中可能失效，尤其当支撑证据与查询缺乏直接重叠、需要中间演绎步骤时。
3. **超参数敏感**：作为无需训练的方法，BEAVER 依赖预设超参数（$M=64$、$\gamma=0.7$、$\lambda=0.7$）。在跨领域迁移时，这些参数可能需要手动调整，不像端到端学习模型那样自动适应。
4. **安全约束丢失风险**：任何压缩方法都可能过滤掉原始上下文中的关键限定词（如否定词、警告语）。在法律或医疗等高敏感领域，此类遗漏可能导致事实错误或有害输出。尽管句子平滑有助于缓解碎片化，用户仍需谨慎并在高风险决策场景中验证输出。

## 我的笔记

BEAVER 的核心价值在于将提示压缩从传统的词元级剪枝重新定义为结构感知的层次化选择，并通过无需训练的方式实现了工业级效率。其页面张量设计不仅提升了 GPU 利用率，也为后续在 RAG（检索增强生成）管道中的集成提供了天然接口。例如，可将检索到的文档块直接作为页面输入 BEAVER，再压缩后送入生成模型。

从复现角度，PageEncoder 的 ITF 权重计算与双路池化实现简单，不依赖额外训练数据，这对资源有限的团队极具吸引力。然而，QueryPlanner 中的三重结构先验（尤其是锚点与流式）需要针对具体任务进行超参调优，这在实际部署中可能增加工程成本。未来方向可包括：在线自适应超参搜索，根据输入领域自动调整 $\gamma$ 与 $\lambda$；混合粒度策略，在页面选择后引入轻量级词元级二次剪枝，进一步消除冗余；将 BEAVER 与推理时扩展（如思维链）结合，验证其在多跳推理中的增强潜力。
